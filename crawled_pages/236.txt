<h1 id="firstHeading" class="firstHeading" lang="en">Z39.50</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>

<p><b>Z39.50</b> is an <a href="/wiki/International_standard" title="International standard">international standard</a> <a href="/wiki/Client%E2%80%93server" class="mw-redirect" title="Client–server">client–server</a>, <a href="/wiki/Application_layer" title="Application layer">application layer</a> <a href="/wiki/Communications_protocol" class="mw-redirect" title="Communications protocol">communications protocol</a> for searching and retrieving information from a <a href="/wiki/Database" title="Database">database</a> over a <a href="/wiki/TCP/IP" class="mw-redirect" title="TCP/IP">TCP/IP</a> <a href="/wiki/Computer_network" title="Computer network">computer network</a>. It is covered by <a href="/wiki/American_National_Standards_Institute" title="American National Standards Institute">ANSI</a>/<a href="/wiki/NISO" class="mw-redirect" title="NISO">NISO</a> standard Z39.50, and <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a> standard 23950. The standard's maintenance agency is the <a href="/wiki/Library_of_Congress" title="Library of Congress">Library of Congress</a>.
</p><p>Z39.50 is widely used<sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items" title="Wikipedia:Manual of Style/Dates and numbers"><span title="The time period mentioned near this tag is ambiguous. (May 2018)">when?</span></a></i>&#93;</sup> in <a href="/wiki/Library" title="Library">library</a> environments, often incorporated into <a href="/wiki/Integrated_library_system" title="Integrated library system">integrated library systems</a> and personal <a href="/wiki/Reference_management_software" title="Reference management software">bibliographic reference software</a>. Interlibrary catalogue searches for <a href="/wiki/Interlibrary_loan" title="Interlibrary loan">interlibrary loan</a> are often implemented with Z39.50 queries.
</p><p>Work on the Z39.50 protocol began in the 1970s, and led to successive versions in 1988, 1992, 1995 and 2003. The <a href="/wiki/Contextual_Query_Language" title="Contextual Query Language">Contextual Query Language</a> (formerly called the Common Query Language)<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> is based on Z39.50 semantics.
</p>

<h2><span class="mw-headline" id="Search_syntax">Search syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Z39.50&amp;action=edit&amp;section=1" title="Edit section: Search syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The protocol supports search, retrieval, sort, and browse. Search queries contain attributes, typically from the bib-1 attribute set which defines six attributes to specify information searches on the server computer: use, relation, position, structure, truncation, completeness. The syntax of Z39.50 allows for very complex queries.
</p><p>In practice, the functional complexity is limited by the uneven implementations by developers and commercial vendors. The syntax of Z39.50 is abstracted from the underlying database structure. For example, if the client specifies an author search using attribute 1003, the server must determine how to map that search to the indexes it contains. This allows Z39.50 queries to be formulated without knowing anything about the target database, but it also means that results for the same query can vary widely among different servers. One server may have an author index and another may use its index of personal names, whether they are authors or not. A third may have no name index and fall back on its keyword index, and yet another may have no suitable index and return an error.
</p><p>An attempt to remedy the inconsistency is the <a href="/wiki/Bath_Profile" title="Bath Profile">Bath Profile</a> (named after <a href="/wiki/Bath,_Somerset" title="Bath, Somerset">Bath</a>, England, where the <a href="/wiki/Working_group" title="Working group">working group</a> first met in 1999). This document rigidly specifies the search syntax to employ for common bibliographic searches, and the expected response of Bath-compliant servers. Implementation of the Bath Profile has been slow but is gradually improving the Z39.50 landscape<sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items" title="Wikipedia:Manual of Style/Dates and numbers"><span title="The time period mentioned near this tag is ambiguous. (May 2018)">when?</span></a></i>&#93;</sup>. The Bath Profile is maintained by <a href="/wiki/Library_and_Archives_Canada" title="Library and Archives Canada">Library and Archives Canada</a>.
</p>
<h2><span class="mw-headline" id="Modernization_efforts">Modernization efforts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Z39.50&amp;action=edit&amp;section=2" title="Edit section: Modernization efforts">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Z39.50 is a pre-<a href="/wiki/World_Wide_Web" title="World Wide Web">Web</a> technology, and various working groups are attempting to update it to fit better into the modern environment. These attempts fall under the designation ZING (Z39.50 International: Next Generation), and pursue various strategies.
</p><p>The successors to Z39.50 are the twin protocols <a href="/wiki/Search/Retrieve_via_URL" title="Search/Retrieve via URL">SRU</a>/<a href="/wiki/Search/Retrieve_Web_Service" title="Search/Retrieve Web Service">SRW</a> (<i>Search/Retrieve via URL</i>/<i>Search/Retrieve Web service</i>), which drop the Z39.50 communications protocol (replacing it with <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a>) while still attempting to preserve the benefits of the query syntax. SRU is <a href="/wiki/Representational_State_Transfer" class="mw-redirect" title="Representational State Transfer">REST</a>-based, and enables queries to be expressed in <a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">URL</a> query strings; SRW uses <a href="/wiki/SOAP_(protocol)" class="mw-redirect" title="SOAP (protocol)">SOAP</a>. Both expect search results to be returned as <a href="/wiki/XML" title="XML">XML</a>.
</p><p>These projects have a much lower barrier to entry for developers than the original Z39.50 protocol<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (September 2010)">citation needed</span></a></i>&#93;</sup>, allowing the relatively small market for library software to benefit from the <a href="/wiki/Web_service" title="Web service">web service</a> tools developed for much larger markets.
</p><p>Alternatives include the following.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup>
</p>
<ul><li>Open Archives Initiative <a href="/wiki/Protocol_for_Metadata_Harvesting" class="mw-redirect" title="Protocol for Metadata Harvesting">Protocol for Metadata Harvesting</a> (OAI-PMH)</li>
<li><a href="/wiki/SPARQL" title="SPARQL">SPARQL</a></li></ul>
<h2><span class="mw-headline" id="Firewall_Information">Firewall Information</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Z39.50&amp;action=edit&amp;section=3" title="Edit section: Firewall Information">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The registered internet <a href="/wiki/IP_port" class="mw-redirect" title="IP port">IP port</a> for Z39.50 is 210. Although the majority of servers use this port, there are dozens of other port numbers used worldwide by Z39.50 (e.g. 2100, 2200, 2210, 2213, 3520, or in one case, ports 2101 and higher for different databases).<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup><sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>

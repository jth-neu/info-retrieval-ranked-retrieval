<h1 id="firstHeading" class="firstHeading" lang="en">Web archiving</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
				<a class="mw-jump-link" href="#p-search">Jump to search</a>
				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><div role="note" class="hatnote navigation-not-searchable">"Web archive" redirects here. For web.archive.org, see <a href="/wiki/Wayback_Machine" title="Wayback Machine">Wayback Machine</a>. For the Safari file format, see <a href="/wiki/Webarchive" title="Webarchive">webarchive</a>. For the Sun Microsystems file format, see <a href="/wiki/WAR_(Sun_file_format)" class="mw-redirect" title="WAR (Sun file format)">WAR (Sun file format)</a>.</div>
<p><b>Web archiving</b> is the process of collecting portions of the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> to ensure the information is <a href="/wiki/Digital_preservation" title="Digital preservation">preserved</a> in an <a href="/wiki/Archive" title="Archive">archive</a> for future researchers, historians, and the public.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> Web archivists typically employ <a href="/wiki/Web_crawler" title="Web crawler">web crawlers</a> for automated capture due to the massive size and amount of information on the Web.  The largest web archiving organization based on a bulk crawling approach is the <a href="/wiki/Internet_Archive" title="Internet Archive">Internet Archive</a> which strives to maintain an archive of the entire Web. 
</p><p>The International Web Archiving Workshop (IWAW), begun in 2001, has provided a platform to share experiences and exchange ideas. The later founding of the International Internet Preservation Consortium (IIPC), in 2003, has greatly facilitated international collaboration in developing standards and open source tools for the creation of web archives. These developments, and the growing portion of human culture created and recorded on the web, combine to make it inevitable that more and more libraries and archives will have to face the challenges of web archiving.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup><a href="/wiki/National_library" title="National library">National libraries</a>, <a href="/wiki/National_archive" class="mw-redirect" title="National archive">national archives</a> and various consortia of organizations are also involved in archiving culturally important Web content. 
</p><p>Commercial web archiving software and services are also available to organizations who need to archive their own web content for corporate heritage, regulatory, or legal purposes.
</p>

<h2><span class="mw-headline" id="Provenance_and_development">Provenance and development</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=1" title="Edit section: Provenance and development">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Early practice of web archiving involved the highlighting of a “site of the week award” as a record for the contest. Besides that, another early practice was the professional link list (for example <a href="/wiki/Amnesty_International" title="Amnesty International">Amnesty International</a>’s list of human rights groups) and (<a rel="nofollow" class="external text" href="https://www.yahoo.com">Yahoo!</a>) directory and Open Directory Project.
</p><p>In the mid-1990s, one of the more important listing sites of its kind continually updated an index of worthwhile website destinations organized by content category.
</p><p>In 1998, Yahoo directory was considered to have made a significant contribution to emerging online library science, not only by its classification scheme but also by the means of the content “navigation” it developed.
</p><p>Soliciting, evaluating, and categorizing websites – the large-scale collecting, hand-sorting, and display of websites – could be considered an original form of website analysis. The rise of the algorithmic search engine has largely led to the disappearance of such manual methods.
</p>
<h2><span class="mw-headline" id="Collecting_the_web">Collecting the web</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=2" title="Edit section: Collecting the web">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Web archivists generally archive various types of web content including <a href="/wiki/HTML" title="HTML">HTML</a> web pages, <a href="/wiki/Style_sheet_(web_development)" title="Style sheet (web development)">style sheets</a>, <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>, <a href="/wiki/Digital_image" title="Digital image">images</a>, and <a href="/wiki/Digital_video" title="Digital video">video</a>. They also archive <a href="/wiki/Metadata" title="Metadata">metadata</a> about the collected resources such as access time, <a href="/wiki/MIME_type" class="mw-redirect" title="MIME type">MIME type</a>, and content length.  This metadata is useful in establishing <a href="/wiki/Authentication" title="Authentication">authenticity</a> and <a href="/wiki/Provenance" title="Provenance">provenance</a> of the archived collection.
</p>
<h2><span class="mw-headline" id="Methods_of_collection">Methods of collection</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=3" title="Edit section: Methods of collection">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/List_of_Web_archiving_initiatives" title="List of Web archiving initiatives">List of Web archiving initiatives</a></div>
<h3><span class="mw-headline" id="Remote_harvesting">Remote harvesting</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=4" title="Edit section: Remote harvesting">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The most common web archiving technique uses <a href="/wiki/Web_crawler" title="Web crawler">web crawlers</a> to automate the process of collecting <a href="/wiki/Web_page" title="Web page">web pages</a>.  Web crawlers typically access web pages in the same manner that users with a browser see the Web, and therefore provide a comparatively simple method of remote harvesting web content. Examples of web crawlers used for web archiving include:
</p>
<ul><li><a href="/wiki/Heritrix" title="Heritrix">Heritrix</a></li>
<li><a href="/wiki/HTTrack" title="HTTrack">HTTrack</a></li>
<li><a href="/wiki/Wget" title="Wget">Wget</a></li></ul>
<p>There exist various free services which may be used to archive web resources "on-demand", using web crawling techniques. These services include the <a href="/wiki/Wayback_Machine" title="Wayback Machine">Wayback Machine</a> and <a href="/wiki/WebCite" title="WebCite">WebCite</a>.
</p>
<h3><span class="mw-headline" id="Database_archiving">Database archiving</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=5" title="Edit section: Database archiving">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Database archiving refers to methods for archiving the underlying content of database-driven websites. It typically requires the extraction of the <a href="/wiki/Database" title="Database">database</a> content into a standard <a href="/wiki/Logical_schema" class="mw-redirect" title="Logical schema">schema</a>, often using <a href="/wiki/XML" title="XML">XML</a>. Once stored in that standard format, the archived content of multiple databases can then be made available using a single access system. This approach is exemplified by the <a rel="nofollow" class="external text" href="http://deeparc.sourceforge.net/">DeepArc</a> and <a rel="nofollow" class="external text" href="https://web.archive.org/web/20110227202744/http://www.nla.gov.au/xinq/">Xinq</a> tools developed by the <a href="/wiki/Biblioth%C3%A8que_nationale_de_France" title="Bibliothèque nationale de France">Bibliothèque nationale de France</a> and the <a href="/wiki/National_Library_of_Australia" title="National Library of Australia">National Library of Australia</a> respectively. DeepArc enables the structure of a <a href="/wiki/Relational_database" title="Relational database">relational database</a> to be mapped to an <a href="/wiki/XML_schema" title="XML schema">XML schema</a>, and the content exported into an XML document. Xinq then allows that content to be delivered online. Although the original layout and behavior of the website cannot be preserved exactly, Xinq does allow the basic querying and retrieval functionality to be replicated.
</p>
<h3><span class="mw-headline" id="Transactional_archiving">Transactional archiving</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=6" title="Edit section: Transactional archiving">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Transactional archiving is an event-driven approach, which collects the actual transactions which take place between a <a href="/wiki/Web_server" title="Web server">web server</a> and a <a href="/wiki/Web_browser" title="Web browser">web browser</a>. It is primarily used as a means of preserving evidence of the content which was actually viewed on a particular <a href="/wiki/Website" title="Website">website</a>, on a given date. This may be particularly important for organizations which need to comply with legal or regulatory requirements for disclosing and retaining information.
</p><p>A transactional archiving system typically operates by intercepting every <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a> request to, and response from, the web server, filtering each response to eliminate duplicate content, and permanently storing the responses as bitstreams.
</p>
<h2><span class="mw-headline" id="Difficulties_and_limitations">Difficulties and limitations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=7" title="Edit section: Difficulties and limitations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Crawlers">Crawlers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=8" title="Edit section: Crawlers">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Web archives which rely on web crawling as their primary means of collecting the Web are influenced by the difficulties of web crawling:
</p>
<ul><li>The <a href="/wiki/Robots_exclusion_protocol" class="mw-redirect" title="Robots exclusion protocol">robots exclusion protocol</a> may request crawlers not access portions of a website.  Some web archivists may ignore the request and crawl those portions anyway.</li>
<li>Large portions of a web site may be hidden in the <a href="/wiki/Deep_Web_(search_indexing)" class="mw-redirect" title="Deep Web (search indexing)">Deep Web</a>.  For example, the results page behind a web form can lie in the Deep Web if crawlers cannot follow a link to the results page.</li>
<li><a href="/wiki/Crawler_trap" class="mw-redirect" title="Crawler trap">Crawler traps</a> (e.g., calendars) may cause a crawler to download an infinite number of pages, so crawlers are usually configured to limit the number of dynamic pages they crawl.</li></ul>
<p>However, it is important to note that a native format web archive, i.e., a fully browsable web archive, with working links, media, etc., is only really possible using crawler technology.
</p><p>The Web is so large that crawling a significant portion of it takes a large amount of technical resources.  The Web is changing so fast that portions of a website may change before a crawler has even finished crawling it.
</p>
<h3><span class="mw-headline" id="General_limitations">General limitations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=9" title="Edit section: General limitations">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Some web servers are configured to return different pages to web archiver requests than they would in response to regular browser requests.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup>  This is typically done to fool search engines into directing more user traffic to a website, and is often done to avoid accountability, or to provide enhanced content only to those browsers that can display it.
</p><p>Not only must web archivists deal with the technical challenges of web archiving, they must also contend with intellectual property laws.  Peter Lyman<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> states that "although the Web is popularly regarded as a <a href="/wiki/Public_domain" title="Public domain">public domain</a> resource, it is <a href="/wiki/Copyright" title="Copyright">copyrighted</a>; thus, archivists have no legal right to copy the Web". However <a href="/wiki/National_library" title="National library">national libraries</a> in some countries <sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup> have a legal right to copy portions of the web under an extension of a <a href="/wiki/Legal_deposit" title="Legal deposit">legal deposit</a>.
</p><p>Some private non-profit web archives that are made publicly accessible like <a href="/wiki/WebCite" title="WebCite">WebCite</a>, the <a href="/wiki/Internet_Archive" title="Internet Archive">Internet Archive</a> or the <a href="/wiki/Internet_Memory_Foundation" title="Internet Memory Foundation">Internet Memory Foundation</a> allow content owners to hide or remove archived content that they do not want the public to have access to. Other web archives are only accessible from certain locations or have regulated usage. WebCite cites  a recent lawsuit against Google's caching, which <a href="/wiki/Google" title="Google">Google</a> won.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Aspects_of_web_curation">Aspects of web curation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_archiving&amp;action=edit&amp;section=10" title="Edit section: Aspects of web curation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Web curation, like any digital curation, entails:
</p>
<ul><li>Certification of the trustworthiness and integrity of the collection content</li>
<li>Collecting verifiable Web assets</li>
<li>Providing Web asset search and retrieval</li>
<li>Semantic and ontological continuity and comparability of the collection content</li></ul>
<p>Thus, besides the discussion on methods of collecting the Web, those of providing access, certification, and organizing must be included. There are a set of popular tools that addresses these curation steps:
</p><p>A suite of tools for Web Curation by <a href="/wiki/International_Internet_Preservation_Consortium" title="International Internet Preservation Consortium">International Internet Preservation Consortium</a>:
</p>
<ul><li><a href="/wiki/Heritrix" title="Heritrix">Heritrix</a> – collecting Web asset</li>
<li><a href="/w/index.php?title=NutchWAX&amp;action=edit&amp;redlink=1" class="new" title="NutchWAX (page does not exist)">NutchWAX</a> – search Web archive collections</li>
<li><a href="/w/index.php?title=Open_source_Wayback_Machine&amp;action=edit&amp;redlink=1" class="new" title="Open source Wayback Machine (page does not exist)">Open source Wayback Machine</a> – search and navigate Web archive collections using NutchWax</li>
<li><a href="/wiki/International_Internet_Preservation_Consortium#Web_Curator_Tool" title="International Internet Preservation Consortium">Web Curator Tool</a> – Selection and Management of Web Collection<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup></li></ul>
<p>Other open source tools for manipulating web archives:
</p>
<ul><li><a href="/w/index.php?title=WARC_Tools&amp;action=edit&amp;redlink=1" class="new" title="WARC Tools (page does not exist)">WARC Tools</a> – for creating, reading, parsing and manipulating, <a href="/wiki/Web_ARChive" title="Web ARChive">WARC</a> archives programmatically</li>
<li>Google Search Tools – for indexing and searching full-text and metadata within web archives<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2014)">citation needed</span></a></i>&#93;</sup></li></ul>
<p>Free but not open source tools also exist:
</p>
<ul><li><a href="/w/index.php?title=WARC_Software_Development_Kit&amp;action=edit&amp;redlink=1" class="new" title="WARC Software Development Kit (page does not exist)">WARC Software Development Kit</a> (WSDK) represents a set of simple, compact, and highly optimized Erlang modules to manipulate (create/read/write) the <a href="/wiki/Web_ARChive" title="Web ARChive">WARC</a> ISO 28500:2009 file format.</li></ul>

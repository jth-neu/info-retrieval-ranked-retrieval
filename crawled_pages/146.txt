<h1 id="firstHeading" class="firstHeading" lang="en">Web development tools</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
				<a class="mw-jump-link" href="#p-search">Jump to search</a>
				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Web development tools</b> allow <a href="/wiki/Web_developers" class="mw-redirect" title="Web developers">web developers</a> to test and <a href="/wiki/Debug" class="mw-redirect" title="Debug">debug</a> their code. They are different from <a href="/wiki/Website_builder" title="Website builder">website builders</a> and <a href="/wiki/Integrated_development_environment" title="Integrated development environment">integrated development environments</a> (IDEs) in that they do not assist in the direct creation of a webpage, rather they are tools used for testing the user interface of a website or web application.
</p><p>Web development tools come as browser add-ons or built-in features in <a href="/wiki/Web_browser" title="Web browser">web browsers</a>. Most popular web browsers, such as <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a>, <a href="/wiki/Firefox" title="Firefox">Firefox</a>, <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a>, <a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a> and <a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a>,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> have built-in tools to help web developers, and many additional add-ons can be found in their respective plugin download centers.
</p><p>Web development tools allow developers to work with a variety of web technologies, including <a href="/wiki/HTML" title="HTML">HTML</a>, <a href="/wiki/CSS" class="mw-redirect" title="CSS">CSS</a>, the <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a>, <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>, and other components that are handled by the web browser. Due to increasing demand from web browsers to do more,<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> popular web browsers have included more features geared for developers.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup>
</p>

<h2><span class="mw-headline" id="Web_developer_tools_support">Web developer tools support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_development_tools&amp;action=edit&amp;section=1" title="Edit section: Web developer tools support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The five most popular web browsers<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> have support for web developer tools that allow web designers and developers to look at the make-up of their pages. These are all tools that are built into the browser and do not require additional modules or configuration.
</p>
<ul><li><a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a> – <i>Web Developer Tools</i><sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup></li>
<li><a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> and <a href="/wiki/Microsoft_Edge" title="Microsoft Edge">Microsoft Edge</a> – <i>F12 Web Developer Tools</i> (as of version 8)<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup><sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup></li>
<li><a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a> – <i>Safari Web Development Tools</i><sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> (as of version 3)<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup></li>
<li><a href="/wiki/Firefox" title="Firefox">Firefox</a> – F12 <i>Web Console / Browser Console</i> (since Firefox 4).<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup><sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup> The Web Console applies to a single content tab; the Browser Console applies to the whole browser.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">&#91;12&#93;</a></sup> Many addons also exist,<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup> including <a href="/wiki/Firebug_(software)" title="Firebug (software)">Firebug</a>.</li>
<li><a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a> – <i><a href="/wiki/Opera_Dragonfly" title="Opera Dragonfly">Opera Dragonfly</a></i><sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup></li></ul>
<h2><span class="mw-headline" id="Most_used_features">Most used features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_development_tools&amp;action=edit&amp;section=2" title="Edit section: Most used features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The built-in web developer tools in the browser are commonly accessed by hovering over an item on a webpage and selecting the "Inspect Element" or similar option from the <a href="/wiki/Context_menu" title="Context menu">context menu</a>. Alternatively the <i>F12</i> key tends to be another common shortcut.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup>
</p>
<h3><span class="mw-headline" id="HTML_and_the_DOM">HTML and the DOM</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_development_tools&amp;action=edit&amp;section=3" title="Edit section: HTML and the DOM">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>HTML and DOM viewer and editor is commonly included in the built-in web development tools. The difference between the HTML and DOM viewer, and the view source feature in web browsers is that the HTML and DOM viewer allows you to see the DOM as it was rendered in addition to allowing you to make changes to the HTML and DOM and see the change reflected in the page after the change is made.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup>
</p><p>In addition to selecting and editing, the HTML elements panels will usually also display properties of the DOM object, such as display dimension, and CSS properties.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">&#91;17&#93;</a></sup>
</p>
<h3><span id="Web_page_assets.2C_resources_and_network_information"></span><span class="mw-headline" id="Web_page_assets,_resources_and_network_information">Web page assets, resources and network information</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_development_tools&amp;action=edit&amp;section=4" title="Edit section: Web page assets, resources and network information">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Web pages typically load and require additional content in the form of images, scripts, font and other external files. Web development tools also allow developers to inspect resources that are loaded and available on the web page in a tree-structure listing.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup><sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup>
</p><p>Web development tools also allow developers to view information about the network usage, such as viewing what the loading time and bandwidth usage are and which <a href="/wiki/HTTP_headers" class="mw-redirect" title="HTTP headers">HTTP headers</a> are being sent and received.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">&#91;20&#93;</a></sup>
</p>
<h3><span class="mw-headline" id="Profiling_and_auditing">Profiling and auditing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_development_tools&amp;action=edit&amp;section=5" title="Edit section: Profiling and auditing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Profiling allows developers to capture information about the performance of a web page or web application. With this information developers can improve the performance of their scripts. Auditing features may provide developers suggestions, after analyzing a page, for optimizations to decrease page load time and increase responsiveness. Web development tools typically also provide a timeline features provides a record of the time it takes to render the page, memory usage, and the types of events which are taking place.<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">&#91;21&#93;</a></sup><sup id="cite_ref-22" class="reference"><a href="#cite_note-22">&#91;22&#93;</a></sup>
</p><p>These features allow developers to optimize their web page or web application.<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">&#91;23&#93;</a></sup>
</p>
<h4><span class="mw-headline" id="Javascript_debugging">Javascript debugging</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_development_tools&amp;action=edit&amp;section=6" title="Edit section: Javascript debugging">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p><a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> is commonly used in web browsers. Web development tools commonly include a panel to debug scripts by allowing developers to add watch expressions, breakpoints, view the call stack, and pause, step over, step into, and step out of functions while debugging JavaScript.
</p><p>A JavaScript console is commonly included. The consoles allow developers to type in JavaScript commands and call functions, or view errors that may have been encountered during the execution of a script.<sup id="cite_ref-24" class="reference"><a href="#cite_note-24">&#91;24&#93;</a></sup><sup id="cite_ref-25" class="reference"><a href="#cite_note-25">&#91;25&#93;</a></sup><sup id="cite_ref-26" class="reference"><a href="#cite_note-26">&#91;26&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Extensions_and_plugins">Extensions and plugins</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_development_tools&amp;action=edit&amp;section=7" title="Edit section: Extensions and plugins">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Modern web browsers support the use of <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plugins</a> or extensions to add or augment functionality.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">&#91;27&#93;</a></sup> There are many common plugins that can provide a diverse range of additional features.
</p>

<h1 id="firstHeading" class="firstHeading" lang="en">Deep web</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
				<a class="mw-jump-link" href="#p-search">Jump to search</a>
				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><div role="note" class="hatnote navigation-not-searchable">This article is about the part of the World Wide Web not indexed by traditional search engines. For other uses, see <a href="/wiki/Deep_web_(disambiguation)" class="mw-disambig" title="Deep web (disambiguation)">Deep web (disambiguation)</a>.</div>
<div role="note" class="hatnote navigation-not-searchable">Not to be confused with <a href="/wiki/Dark_web" title="Dark web">Dark web</a>.</div>
<p>The <b>deep web</b>,<sup id="cite_ref-nhamilton_1-0" class="reference"><a href="#cite_note-nhamilton-1">&#91;1&#93;</a></sup> <b>invisible web</b>,<sup id="cite_ref-jal_2-0" class="reference"><a href="#cite_note-jal-2">&#91;2&#93;</a></sup> or <b>hidden web</b><sup id="cite_ref-cthw_3-0" class="reference"><a href="#cite_note-cthw-3">&#91;3&#93;</a></sup> are parts of the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> whose contents are not <a href="/wiki/Search_engine_indexing" title="Search engine indexing">indexed</a> by standard <a href="/wiki/Web_search_engine" title="Web search engine">web search engines</a> for any reason. The opposite term to the deep web is the <a href="/wiki/Surface_web" title="Surface web">surface web</a>, which is accessible to anyone using the Internet.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>  <a href="/wiki/Computer_science" title="Computer science">Computer scientist</a> Michael K. Bergman is credited with coining the term <i>deep web</i> in 2001 as a search indexing term.<sup id="cite_ref-wright2009_5-0" class="reference"><a href="#cite_note-wright2009-5">&#91;5&#93;</a></sup>
</p><p>The content of the deep web is hidden behind HTTP forms,<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup><sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> and includes many very common uses such as <a href="/wiki/Web_mail" class="mw-redirect" title="Web mail">web mail</a>, <a href="/wiki/Online_banking" title="Online banking">online banking</a>, and services that users must pay for, and which is protected by a <a href="/wiki/Paywall" title="Paywall">paywall</a>, such as <a href="/wiki/Video_on_demand" title="Video on demand">video on demand</a>, some online magazines and newspapers, among others.
</p><p>Content of the deep web can be located and accessed by a direct <a href="/wiki/URL" title="URL">URL</a> or <a href="/wiki/IP_address" title="IP address">IP address</a>, and may require password or other security access past the public website page.
</p>

<h2><span class="mw-headline" id="Terminology">Terminology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Deep_web&amp;action=edit&amp;section=1" title="Edit section: Terminology">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The first <a href="/wiki/Skunked_term" title="Skunked term">conflat</a><sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup><a href="/wiki/Skunked_term" title="Skunked term">ion</a> of the terms "deep web" and "<a href="/wiki/Dark_web" title="Dark web">dark web</a>" based came about in 2009 when the deep web search terminology was discussed alongside illegal activities taking place on the <a href="/wiki/Freenet" title="Freenet">Freenet</a> <a href="/wiki/Darknet" title="Darknet">darin et</a>.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup>
</p><p>Since then, the use in the <a href="/wiki/Silk_Road_(marketplace)" title="Silk Road (marketplace)">Silk Road</a>'s media reporting, many<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup><sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup> people and media outlets, have taken to using Deep Web <a href="/wiki/Skunked_term" title="Skunked term">synonymously</a> with the <a href="/wiki/Dark_web" title="Dark web">dark web</a> or <a href="/wiki/Darknet" title="Darknet">darknet</a>, a comparison many reject as inaccurate<sup id="cite_ref-confusion_12-0" class="reference"><a href="#cite_note-confusion-12">&#91;12&#93;</a></sup> and consequently is an ongoing source of confusion.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup> <i><a href="/wiki/Wired_(website)" class="mw-redirect" title="Wired (website)">Wired</a></i> reporters <a href="/wiki/Kim_Zetter" title="Kim Zetter">Kim Zetter</a><sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup> and <a href="/wiki/Andy_Greenberg" title="Andy Greenberg">Andy Greenberg</a><sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup> recommend the terms be used in distinct fashions. While the deep web is reference to any site that cannot be accessed through a traditional search engine, the dark web is a portion of the deep web that has been intentionally hidden and is inaccessible through standard browsers and methods.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup><sup id="cite_ref-17" class="reference"><a href="#cite_note-17">&#91;17&#93;</a></sup><sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup><sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup><sup id="cite_ref-20" class="reference"><a href="#cite_note-20">&#91;20&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Non-indexed_content">Non-indexed content</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Deep_web&amp;action=edit&amp;section=2" title="Edit section: Non-indexed content">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Bergman, in a paper on the deep web published in <i>The Journal of Electronic Publishing</i>, mentioned that Jill Ellsworth used the term <i>Invisible Web</i> in 1994 to refer to <a href="/wiki/Website" title="Website">websites</a> that were not registered with any search engine.<sup id="cite_ref-bergman2001_21-0" class="reference"><a href="#cite_note-bergman2001-21">&#91;21&#93;</a></sup> Bergman cited a January 1996 article by Frank Garcia:<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">&#91;22&#93;</a></sup>
</p>
<blockquote><p>It would be a site that's possibly reasonably designed, but they didn't bother to register it with any of the search engines. So, no one can find them! You're hidden. I call that the invisible Web.</p></blockquote>
<p>Another early use of the term <i>Invisible Web</i> was by Bruce Mount and Matthew B. Koll of Personal Library Software, in a description of the #1 Deep Web tool found in a December 1996 press release.<sup id="cite_ref-PLS_23-0" class="reference"><a href="#cite_note-PLS-23">&#91;23&#93;</a></sup>
</p><p>The first use of the specific term <i>deep web</i>, now generally accepted, occurred in the aforementioned 2001 Bergman study.<sup id="cite_ref-bergman2001_21-1" class="reference"><a href="#cite_note-bergman2001-21">&#91;21&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Indexing_methods">Indexing methods</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Deep_web&amp;action=edit&amp;section=3" title="Edit section: Indexing methods">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Methods which prevent web pages from being indexed by traditional search engines may be categorized as one or more of the following:
</p>
<ol><li><b>Contextual web</b>: pages with content varying for different access contexts (e.g., ranges of client IP addresses or previous navigation sequence).</li>
<li><b>Dynamic content</b>: <a href="/wiki/Dynamic_Web_page" class="mw-redirect" title="Dynamic Web page">dynamic pages</a> which are returned in response to a submitted query or accessed only through a form, especially if open-domain input elements (such as text fields) are used; such fields are hard to navigate without <a href="/wiki/Domain_knowledge" title="Domain knowledge">domain knowledge</a>.</li>
<li><b>Limited access content</b>: sites that limit access to their pages in a technical way (e.g., using the <a href="/wiki/Robots_Exclusion_Standard" class="mw-redirect" title="Robots Exclusion Standard">Robots Exclusion Standard</a> or <a href="/wiki/CAPTCHA" title="CAPTCHA">CAPTCHAs</a>, or no-store directive which prohibit search engines from browsing them and creating <a href="/wiki/Web_cache" title="Web cache">cached</a> copies).<sup id="cite_ref-24" class="reference"><a href="#cite_note-24">&#91;24&#93;</a></sup></li>
<li><b>Non-HTML/text content</b>: textual content encoded in multimedia (image or video) files or specific <a href="/wiki/File_formats" class="mw-redirect" title="File formats">file formats</a> not handled by search engines.</li>
<li><b>Private web</b>: sites that require registration and login (password-protected resources).</li>
<li><b>Scripted content</b>: pages that are only accessible through links produced by <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> as well as content dynamically downloaded from Web servers via <a href="/wiki/Macromedia_Flash" class="mw-redirect" title="Macromedia Flash">Flash</a> or <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a> solutions.</li>
<li><b>Software</b>: certain content is intentionally hidden from the regular Internet, accessible only with special software, such as <a href="/wiki/Tor_(anonymity_network)" title="Tor (anonymity network)">Tor</a>, <a href="/wiki/I2P" title="I2P">I2P</a>, or other darknet software. For example, Tor allows users to access websites using the <a href="/wiki/.onion" title=".onion">.onion</a> server address anonymously, hiding their IP address.</li>
<li><b>Unlinked content</b>: pages which are not linked to by other pages, which may prevent <a href="/wiki/Web_crawling" class="mw-redirect" title="Web crawling">web crawling</a> programs from accessing the content. This content is referred to as pages without <a href="/wiki/Backlink" title="Backlink">backlinks</a> (also known as inlinks). Also, search engines do not always detect all backlinks from searched web pages.</li>
<li><b>Web archives</b>: Web archival services such as the <a href="/wiki/Wayback_Machine" title="Wayback Machine">Wayback Machine</a> enable users to see archived versions of web pages across time, including websites which have become inaccessible, and are not indexed by search engines such as Google.<sup id="cite_ref-25" class="reference"><a href="#cite_note-25">&#91;25&#93;</a></sup></li></ol>
<h2><span class="mw-headline" id="Content_types">Content types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Deep_web&amp;action=edit&amp;section=4" title="Edit section: Content types">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>While it is not always possible to directly discover a specific web server's content so that it may be indexed, a site potentially can be accessed indirectly (due to <a href="/wiki/Vulnerability_(computing)" title="Vulnerability (computing)">computer vulnerabilities</a>).
</p><p>To discover content on the web, search engines use <a href="/wiki/Web_crawlers" class="mw-redirect" title="Web crawlers">web crawlers</a> that follow hyperlinks through known protocol virtual <a href="/wiki/Port_numbers" class="mw-redirect" title="Port numbers">port numbers</a>. This technique is ideal for discovering content on the surface web but is often ineffective at finding deep web content. For example, these crawlers do not attempt to find dynamic pages that are the result of database queries due to the indeterminate number of queries that are possible.<sup id="cite_ref-wright2009_5-1" class="reference"><a href="#cite_note-wright2009-5">&#91;5&#93;</a></sup> It has been noted that this can be (partially) overcome by providing links to query results, but this could unintentionally inflate the popularity for a member of the deep web.
</p><p><a href="/wiki/DeepPeep" title="DeepPeep">DeepPeep</a>, <a href="/wiki/Intute" title="Intute">Intute</a>, <a href="/wiki/Deep_Web_Technologies" title="Deep Web Technologies">Deep Web Technologies</a>, <a href="/wiki/Scirus" title="Scirus">Scirus</a>, and <a href="/wiki/Ahmia.fi" class="mw-redirect" title="Ahmia.fi">Ahmia.fi</a> are a few search engines that have accessed the deep web. Intute ran out of funding and is now a temporary static archive as of July 2011.<sup id="cite_ref-26" class="reference"><a href="#cite_note-26">&#91;26&#93;</a></sup> Scirus retired near the end of January 2013.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">&#91;27&#93;</a></sup>
</p><p>Researchers have been exploring how the deep web can be crawled in an automatic fashion, including content that can be accessed only by special software such as <a href="/wiki/Tor_(anonymity_network)" title="Tor (anonymity network)">Tor</a>. In 2001, Sriram Raghavan and Hector Garcia-Molina (Stanford Computer Science Department, Stanford University)<sup id="cite_ref-raghavan2000_28-0" class="reference"><a href="#cite_note-raghavan2000-28">&#91;28&#93;</a></sup><sup id="cite_ref-29" class="reference"><a href="#cite_note-29">&#91;29&#93;</a></sup> presented an architectural model for a hidden-Web crawler that used key terms provided by users or collected from the query interfaces to query a Web form and crawl the Deep Web content. Alexandros Ntoulas, Petros Zerfos, and Junghoo Cho of <a href="/wiki/University_of_California,_Los_Angeles" title="University of California, Los Angeles">UCLA</a> created a hidden-Web crawler that automatically generated meaningful queries to issue against search forms.<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">&#91;30&#93;</a></sup> Several form query languages (e.g., DEQUEL<sup id="cite_ref-31" class="reference"><a href="#cite_note-31">&#91;31&#93;</a></sup>) have been proposed that, besides issuing a query, also allow extraction of structured data from result pages. Another effort is <a href="/wiki/DeepPeep" title="DeepPeep">DeepPeep</a>, a project of the <a href="/wiki/University_of_Utah" title="University of Utah">University of Utah</a> sponsored by the <a href="/wiki/National_Science_Foundation" title="National Science Foundation">National Science Foundation</a>, which gathered hidden-web sources (web forms) in different domains based on novel focused crawler techniques.<sup id="cite_ref-32" class="reference"><a href="#cite_note-32">&#91;32&#93;</a></sup><sup id="cite_ref-33" class="reference"><a href="#cite_note-33">&#91;33&#93;</a></sup>
</p><p>Commercial search engines have begun exploring alternative methods to crawl the deep web. The <a href="/wiki/Sitemap_Protocol" class="mw-redirect" title="Sitemap Protocol">Sitemap Protocol</a> (first developed, and introduced by Google in 2005) and <a href="/wiki/Open_Archives_Initiative_Protocol_for_Metadata_Harvesting" title="Open Archives Initiative Protocol for Metadata Harvesting">OAI-PMH</a> are mechanisms that allow search engines and other interested parties to discover deep web resources on particular web servers. Both mechanisms allow web servers to advertise the URLs that are accessible on them, thereby allowing automatic discovery of resources that are not directly linked to the surface web. Google's deep web surfacing system computes submissions for each HTML form and adds the resulting HTML pages into the Google search engine index. The surfaced results account for a thousand queries per second to deep web content.<sup id="cite_ref-34" class="reference"><a href="#cite_note-34">&#91;34&#93;</a></sup> In this system, the pre-computation of submissions is done using three algorithms:
</p>
<ol><li>selecting input values for text search inputs that accept keywords,</li>
<li>identifying inputs which accept only values of a specific type (e.g., date), and</li>
<li>selecting a small number of input combinations that generate URLs suitable for inclusion into the Web search index.</li></ol>
<p>In 2008, to facilitate users of <a href="/wiki/Tor_(anonymity_network)#Hidden_services" title="Tor (anonymity network)">Tor hidden services</a> in their access and search of a hidden <a href="/wiki/.onion" title=".onion">.onion</a> suffix, <a href="/wiki/Aaron_Swartz" title="Aaron Swartz">Aaron Swartz</a> designed <a href="/wiki/Tor2web" title="Tor2web">Tor2web</a>—a proxy application able to provide access by means of common web browsers.<sup id="cite_ref-RELEASE_35-0" class="reference"><a href="#cite_note-RELEASE-35">&#91;35&#93;</a></sup> Using this application, deep web links appear as a random string of letters followed by the .onion TLD.
</p>

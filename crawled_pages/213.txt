<h1 id="firstHeading" class="firstHeading" lang="en">Web portal</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>

<p>A <b>web portal</b> is a specially designed <a href="/wiki/Website" title="Website">website</a> that brings information from diverse sources, like <a href="/wiki/Email" title="Email">emails</a>, <a href="/wiki/Internet_forum" title="Internet forum">online forums</a> and <a href="/wiki/Web_search_engine" title="Web search engine">search engines</a>, together in a uniform way. Usually, each information source gets its dedicated area on the page for displaying information (a <a href="/wiki/Portlet" class="mw-redirect" title="Portlet">portlet</a>); often, the user can configure which ones to display. Variants of portals include <a href="/wiki/Mashup_(web_application_hybrid)" title="Mashup (web application hybrid)">mashups</a> and <a href="/wiki/Intranet" title="Intranet">intranet</a> "dashboards" for executives and managers. The extent to which content is displayed in a "uniform way" may depend on the intended user and the intended purpose, as well as the diversity of the content. Very often design emphasis is on a certain "metaphor" for configuring and customizing the presentation of the content (e.g., a dashboard or map) and the chosen implementation framework or code libraries. In addition, the role of the user in an organization may determine which content can be added to the portal or deleted from the portal configuration.
</p><p>A portal may use a search engine's <a href="/wiki/Application_programming_interface" title="Application programming interface">application programming interface</a> (API) to permit users to search <a href="/wiki/Intranet" title="Intranet">intranet</a> content as opposed to <a href="/wiki/Extranet" title="Extranet">extranet</a> content by restricting which domains may be searched. Apart from this common <a href="/wiki/Search_engines" class="mw-redirect" title="Search engines">search engines</a> feature, web portals may offer other services such as <a href="/wiki/E-mail" class="mw-redirect" title="E-mail">e-mail</a>, news, stock quotes, information from databases and even entertainment content. Portals provide a way for enterprises and organizations to provide a consistent "<a href="/wiki/Look_and_feel" title="Look and feel">look and feel</a>" with access control and procedures for multiple applications and databases, which otherwise would have been different web entities at various <a href="/wiki/URL" title="URL">URLs</a>. The features available may be restricted by whether access is by an authorized and authenticated user (employee, member) or an anonymous website visitor.
</p><p>Examples of early public web portals were <a href="/wiki/AOL" title="AOL">AOL</a>, <a href="/wiki/Excite" title="Excite">Excite</a>, <a href="/wiki/Netvibes" title="Netvibes">Netvibes</a>, <a href="/wiki/IGoogle" title="IGoogle">iGoogle</a>, <a href="/wiki/MSN" title="MSN">MSN</a>, <a href="/wiki/Naver" title="Naver">Naver</a>, <a href="/wiki/Lycos" title="Lycos">Lycos</a>, <a href="/wiki/Prodigy_(online_service)" title="Prodigy (online service)">Prodigy</a>, <a href="/wiki/Indiatimes" class="mw-redirect" title="Indiatimes">Indiatimes</a>, <a href="/wiki/Rediff" class="mw-redirect" title="Rediff">Rediff</a>, and <a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a>. See for example, the "My Yahoo!" feature of Yahoo! that may have inspired such features as the later Google "iGoogle" (discontinued as of November 1, 2013.) The configurable side-panels of, for example, the modern Opera browser and the option of "speed dial" pages by most browsers continue to reflect the earlier "portal" metaphor.
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In the late 1990s the Web portal was a Web IT buzzword. After the proliferation of <a href="/wiki/Web_browser" title="Web browser">Web browsers</a> in the late-1990s many companies tried to build or acquire a portal to attempt to obtain a share of an Internet market. The Web portal gained special attention because it was, for many users, the starting point of their Web browsing if it was set as their home page. The content and branding of a portal could change as Internet companies merged or were acquired. <a href="/wiki/Netscape" title="Netscape">Netscape</a> became a part of <a href="/wiki/America_Online" class="mw-redirect" title="America Online">America Online</a>, the <a href="/wiki/Walt_Disney_Company" class="mw-redirect" title="Walt Disney Company">Walt Disney Company</a> launched <a href="/wiki/Go.com" title="Go.com">Go.com</a>, IBM and others launched <a href="/wiki/Prodigy_(online_service)" title="Prodigy (online service)">Prodigy</a> (-only users.) Portal metaphors are widely used by public library sites for borrowers using a login as users and by university intranets for students and for faculty. Vertical markets remain for ISV's (Independent Software Vendors) offering management and executive intranet "dashboards" for corporations and government agencies in areas such as <a href="/wiki/Governance,_risk_management,_and_compliance" title="Governance, risk management, and compliance">GRC</a> and risk management.
</p>
<h2><span class="mw-headline" id="Classification">Classification</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=2" title="Edit section: Classification">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Web portals are sometimes classified as <i>horizontal</i> or <i>vertical</i>. A horizontal portal is used as a platform to several companies in the same economic sector or to the same type of manufacturers or distributors.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> A vertical portal (also known as a "vortal") is a specialized entry point to a specific market or industry niche, subject area, or interest.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> Some vertical portals are known as "vertical information portals" (VIPs). VIPs provide news, editorial content, digital publications, and e-commerce capabilities. In contrast to traditional vertical portals, VIPs also provide dynamic multimedia applications including social networking, video posting, and blogging.
</p>
<h2><span class="mw-headline" id="Types">Types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=3" title="Edit section: Types">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Personal">Personal</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=4" title="Edit section: Personal">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A personal portal is a Web Page at a Web site on the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> or a local HTML home page including JavaScript and perhaps running in a modified Web browser. A personal portal typically provides personalized capabilities to its visitors or its local user, providing a pathway to other content. It may be designed to use <a href="/wiki/Distributed_application" class="mw-redirect" title="Distributed application">distributed applications</a>, different numbers and types of <a href="/wiki/Middleware_(distributed_applications)" title="Middleware (distributed applications)">middleware</a> and hardware to provide services from a number of different sources and may run on a non-standard local Web server. In addition, business portals can be designed for sharing and collaboration in workplaces. A further business-driven requirement of portals is that the content be presented on multiple platforms such as <a href="/wiki/Personal_computer" title="Personal computer">personal computers</a>, laptops, tablet computers, <a href="/wiki/Personal_digital_assistant" title="Personal digital assistant">personal digital assistants</a> (PDAs), <a href="/wiki/Cell_phone" class="mw-redirect" title="Cell phone">cell phones</a> and <a href="/wiki/Smartphone" title="Smartphone">smartphones</a>.
</p><p>Information, news, and updates are examples of content that could be delivered through such a portal. Personal portals can be related to any specific topic such as providing friends information on a social network or providing links to outside content that may help others beyond your reach of services. Portals are not limited to simply providing links. Outside of business intranet user, very often simpler portals become replaced with richer mashup designs. Within enterprises, early portals were often replaced by much more powerful "dashboard" designs. Some also have relied on newer protocols such as some version of RSS aggregation and may or may not involve some degree of Web harvesting. Facebook can be considered as a modern personal web portal.
</p>
<h3><span class="mw-headline" id="Government">Government</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=5" title="Edit section: Government">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>At the end of the dot-com boom in the 1990s, many governments had already committed to creating government web portal sites for their citizens. These included primary portals to the governments as well as portals developed for specific branches (e.g., a particular government ministry, department or agency), or for specific sub-audiences (e.g., senior citizens, parents, post-secondary students, etc.). Examples of government Web portals include:
</p>
<ul><li><a href="/wiki/Australia.gov.au" title="Australia.gov.au">australia.gov.au</a> for <a href="/wiki/Australia" title="Australia">Australia</a>.</li>
<li><a href="/wiki/USA.gov" title="USA.gov">USA.gov</a> for the United States (in English) &amp; <a href="/wiki/GobiernoUSA.gov" title="GobiernoUSA.gov">GobiernoUSA.gov</a> (in Spanish).</li></ul>
<ul><li>www.gov.lk for <a href="/wiki/Sri_Lanka" title="Sri Lanka">Sri Lanka</a>.</li>
<li><a href="/wiki/Disability.gov" title="Disability.gov">Disability.gov</a> for citizens with disabilities in the United States.</li>
<li><a href="/wiki/Europa_(web_portal)" title="Europa (web portal)">Europa (web portal)</a> links to all EU agencies and institutions in addition to press releases and audiovisual content from press conferences.</li>
<li><a href="/wiki/Gov.uk" title="Gov.uk">gov.uk</a> for citizens &amp; <a href="/wiki/Businesslink.gov.uk" class="mw-redirect" title="Businesslink.gov.uk">businesslink.gov.uk</a> for businesses in the United Kingdom.</li>
<li><a href="/wiki/Health-EU_portal" title="Health-EU portal">Health-EU portal</a> gathers all relevant health topics from across Europe.</li>
<li><a href="/wiki/India.gov.in" title="India.gov.in">india.gov.in</a> for <a href="/wiki/India" title="India">India</a>.</li>
<li><a href="/wiki/National_Resource_Directory" title="National Resource Directory">National Resource Directory</a> links to resources for United States Service Members, Veterans and their families.</li>
<li>govt.nz for <a href="/wiki/New_Zealand" title="New Zealand">New Zealand</a>.</li>
<li>Saudi.gov.sa for Saudi Arabia.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup></li>
<li><a rel="nofollow" class="external free" href="https://online.belastingdienst.cw">https://online.belastingdienst.cw</a> for the Tax Authority of <a href="/wiki/Cura%C3%A7ao" title="Curaçao">Curaçao</a> (part of the Kingdom of the Netherlands)</li></ul>
<h3><span class="mw-headline" id="Cultural">Cultural</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=6" title="Edit section: Cultural">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Cultural portals aggregate digitised cultural collections of galleries, libraries (see: <a href="/wiki/Library_portal" title="Library portal">library portal</a>), archives and museums. This type of portal provides a point of access to <a href="/wiki/Deep_Web_(search_indexing)" class="mw-redirect" title="Deep Web (search indexing)">invisible Web</a> cultural content that may not be indexed by standard search engines. Digitised collections can include scans or digital photos of books, artworks, photography, journals, newspapers, maps, diaries and letters and digital files of music, sound recordings, films, and archived websites as well as the descriptive <a href="/wiki/Metadata" title="Metadata">metadata</a> associated with each type of cultural work (e.g., metadata provides information about the author, publisher, etc.). These portals are often based around a specific national or regional groupings of institutions. Examples of cultural portals include:
</p>
<ul><li><a href="/wiki/DigitalNZ" title="DigitalNZ">DigitalNZ</a> – A cultural portal led by the National Library of New Zealand focused on New Zealand digital content.</li>
<li><a href="/wiki/Europeana" title="Europeana">Europeana</a> – A cultural portal for the European Union based in the National Library of the Netherlands and overseen by the Europeana Foundation.</li>
<li>Trove – A cultural portal led by the National Library of Australia focused on Australian content.</li>
<li><a href="/wiki/TUT.by" class="mw-redirect" title="TUT.by">TUT.by</a> - A commercial cultural portal focused on Belarusian digital content.</li>
<li><a href="/wiki/Digital_Public_Library_of_America" title="Digital Public Library of America">Digital Public Library of America</a> (in development)</li></ul>
<h3><span class="mw-headline" id="Corporate">Corporate</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=7" title="Edit section: Corporate">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Intranet_portal" title="Intranet portal">Intranet portal</a></div>
<p>Corporate intranets became common during the 1990s. As intranets grew in size and complexity, organization webmasters were faced with increasing content and user management challenges. A consolidated view of company information was judged insufficient; users wanted personalization and customization. Webmasters, if skilled enough, were able to offer some capabilities, but for the most part ended up driving users away from using the intranet. Many companies began to offer tools to help webmasters manage their data, applications and information more easily, and by providing different users with personalized views. Portal solutions can also include workflow management, collaboration between work groups or branches, and policy-managed content publication. Most can allow internal and external access to specific corporate information using secure authentication or <a href="/wiki/Single_sign-on" title="Single sign-on">single sign-on</a>.
</p><p>JSR168 Standards emerged around 2001. Java Specification Request (JSR) 168 standards allow the interoperability of <a href="/wiki/Portlets" class="mw-redirect" title="Portlets">portlets</a> across different portal platforms. These standards allow portal developers, administrators and consumers to integrate standards-based portals and portlets across a variety of vendor solutions. The concept of content aggregation seems to still gain momentum and portal solution will likely continue to evolve significantly over the next few years. The <a href="/wiki/Gartner_Group" class="mw-redirect" title="Gartner Group">Gartner Group</a> predicts generation 8 portals to expand on the <a href="/wiki/Business_Mashups" class="mw-redirect" title="Business Mashups">Business Mashups</a> concept of delivering a variety of information, tools, applications and access points through a single mechanism.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2009)">citation needed</span></a></i>&#93;</sup>
</p><p>With the increase in user-generated content (blog posts, comments, photos), disparate data silos, and file formats, information architects and taxonomists will be required to allow users the ability to tag (classify) the data or content. For example, if a vice-president makes a blog post, this post could be tagged with her/his name, title, and the subject of the post. Tagging makes it easier for users of the intranet to find the content they are interested in. This will ultimately cause a ripple effect where users will also be generating ad hoc navigation and information flows. Corporate portals also offer customers and employees self-service opportunities.
</p>
<h3><span class="mw-headline" id="Stock">Stock</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=8" title="Edit section: Stock">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Also known as stock-share portals, <a href="/wiki/Stock_market" title="Stock market">stock market</a> portals or <a href="/wiki/Stock_exchange" title="Stock exchange">stock exchange</a> portals are Web-based applications that facilitates the process of informing the share-holders with substantial online data such as the latest price, ask/bids, the latest News, reports and announcements. Some stock portals use online gateways through a central depository system (CDS) for the visitors (ram) to buy or sell their shares or manage their <a href="/wiki/Portfolio_(finance)" title="Portfolio (finance)">portfolio</a>.
</p>
<h3><span class="mw-headline" id="Search">Search</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=9" title="Edit section: Search">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Search portals aggregate results from several search engines into one page. You can find search portals specialized in a product, for example property search portals. Library search portals are also known as discovery interfaces.
</p>
<h4><span class="mw-headline" id="Property_search">Property search</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=10" title="Edit section: Property search">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Property search portals aggregate data about properties for sale by real estate agents. Examples in the UK include <a href="/wiki/Zoopla" class="mw-redirect" title="Zoopla">Zoopla</a>, <a href="/wiki/Rightmove" title="Rightmove">Rightmove</a>, <a href="/wiki/Nestoria" title="Nestoria">Nestoria</a> and <a href="/wiki/Nuroa" title="Nuroa">Nuroa</a>. Examples in the US include Propertini.
</p>
<h3><span class="mw-headline" id="Tender">Tender</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=11" title="Edit section: Tender">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A tender portal is a gateway for government suppliers to bid on providing goods and services. Tender portals allow users to search, modify, submit, review and archive data in order to provide a complete online tendering process.
</p><p>Using online tendering, bidders can do any of the following:
</p>
<ul><li>Receive notification of the tenders.</li>
<li>Receive tender documents online.</li>
<li>Fill out the forms online.</li>
<li>Submit proposals and documents.</li>
<li>Submit bids online.</li></ul>
<h3><span class="mw-headline" id="Hosted">Hosted</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=12" title="Edit section: Hosted">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Hosted Web portals gained popularity and a number of companies began offering them as a hosted service. The hosted portal market fundamentally changed the composition of portals. In many ways they served simply as a tool for publishing information instead of the loftier goals of integrating legacy applications or presenting correlated data from distributed databases. The early hosted portal companies such as <a href="/wiki/Hyperoffice" title="Hyperoffice">Hyperoffice.com</a> or the now defunct InternetPortal.com focused on collaboration and scheduling in addition to the distribution of corporate data. As hosted Web portals have risen in popularity their feature set has grown to include hosted databases, document management, email, discussion forums and more. Hosted portals automatically personalize the content generated from their modules to provide a personalized experience to their users. In this regard they have remained true to the original goals of the earlier corporate Web portals. 
</p><p>Emerging new classes of Internet portals called Cloud Portals are showcasing the power of API (Application Programming Interface) rich software systems leveraging SOA (service-oriented architecture, Web services, and custom data exchange) to accommodate machine to machine interaction creating a more fluid user experience for connecting users spanning multiple domains during a given "session". Cloud portals like <a href="/w/index.php?title=Nubifer_Cloud_Portal&amp;action=edit&amp;redlink=1" class="new" title="Nubifer Cloud Portal (page does not exist)">Nubifer Cloud Portal</a> show what is possible using Enterprise Mashup and Web Service integration approaches to building cloud portals.
</p>
<h3><span class="mw-headline" id="Domain-specific">Domain-specific</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=13" title="Edit section: Domain-specific">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A number of portals have come about which are specific to a particular domain, offering access to related companies and services; a prime example of this trend would be the growth in property portals that give access to services such as <a href="/wiki/Estate_agent" title="Estate agent">estate agents</a>, <a href="/wiki/Moving_company" title="Moving company">removal firm</a>, and <a href="/wiki/Solicitor" title="Solicitor">solicitors</a> that offer <a href="/wiki/Licensed_Conveyancer" class="mw-redirect" title="Licensed Conveyancer">conveyancing</a>. Along the same lines, industry-specific news and information portals have appeared, such as the <a href="/wiki/Clinical_Trial_Portal" class="mw-redirect" title="Clinical Trial Portal">clinical trials-specific portal</a>.
</p>
<h2><span class="mw-headline" id="Engineering_aspects">Engineering aspects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=14" title="Edit section: Engineering aspects">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_portal&amp;action=edit&amp;section=15" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The main concept is to present the user with a single Web page that brings together or aggregates content from a number of other systems or servers. The application server or architecture performs most of the crucial functions of the application. This application server is in turn connected to database servers, and may be part of a clustered server environment. High-capacity portal configurations may include load balancing strategies. For portals that present application functionality to the user, the portal server is in reality the front piece of a server configuration that includes some connectivity to the application server. For early Web browsers permitting HTML frameset and iframe elements, diverse information could be presented without violating the browser same-source security policy (relied upon to prevent a variety of cross-site security breaches). More recent client-side technologies rely on JavaScript frameworks and libraries that rely on more recent Web functionality such as <a href="/wiki/WebSocket" title="WebSocket">WebSockets</a> and asynchronous callbacks using <a href="/wiki/XMLHttpRequest" title="XMLHttpRequest">XMLHttpRequests</a>.
</p><p>The server hosting the portal may only be a "pass through" for the user. By use of <a href="/wiki/Portlet" class="mw-redirect" title="Portlet">portlets</a>, application functionality can be presented in any number of portal pages. For the most part, this architecture is transparent to the user. In such a design, security and concurrent user capacity can be important issues, and security designers need to ensure that only authenticated and authorized users can generate requests to the application server. If the security design and administration does not ensure adequate authentication and authorization, then the portal may inadvertently present vulnerabilities to various types of attacks.
</p>

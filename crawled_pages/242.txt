<h1 id="firstHeading" class="firstHeading" lang="en">Site map</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"><span class="mw-redirectedfrom">  (Redirected from <a href="/w/index.php?title=Sitemap&amp;redirect=no" class="mw-redirect" title="Sitemap">Sitemap</a>)</span></div>

<div role="note" class="hatnote navigation-not-searchable"><span class="plainlinks selfreference noprint">For Wikipedia site map, see <a href="/wiki/Wikipedia:Directory" class="mw-redirect" title="Wikipedia:Directory">Wikipedia:Directory</a>.</span></div>
<p>A <b>site map</b> (or <b>sitemap</b>) is a list of <a href="/wiki/Web_page" title="Web page">pages</a> of a <a href="/wiki/Web_site" class="mw-redirect" title="Web site">web site</a>.
</p><p>There are three primary kinds of site map:
</p>
<ul><li>Site maps used during the planning of a Web site by its <a href="/wiki/Web_design" title="Web design">designers</a>.</li>
<li>Human-visible listings, typically hierarchical, of the pages on a site.</li>
<li>Structured listings intended for <a href="/wiki/Web_crawler" title="Web crawler">web crawlers</a> such as <a href="/wiki/Web_search_engine" title="Web search engine">search engines</a>.</li></ul>

<h2><span class="mw-headline" id="Types_of_site_maps">Types of site maps</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Site_map&amp;action=edit&amp;section=1" title="Edit section: Types of site maps">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright"><div class="thumbinner" style="width:502px;"><a href="/wiki/File:Main_Page_Usability.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/83/Main_Page_Usability.png/500px-Main_Page_Usability.png" width="500" height="271" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/83/Main_Page_Usability.png/750px-Main_Page_Usability.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/83/Main_Page_Usability.png/1000px-Main_Page_Usability.png 2x" data-file-width="1917" data-file-height="1040" /></a>  <div class="thumbcaption"><div class="magnify"><a href="/wiki/File:Main_Page_Usability.png" class="internal" title="Enlarge"></a></div>A site map of what links from the <a href="/wiki/English_Wikipedia" title="English Wikipedia">English Wikipedia</a>'s Main Page.</div></div></div>
<div class="thumb tright"><div class="thumbinner" style="width:402px;"><a href="/wiki/File:Sitemap_google.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/20/Sitemap_google.jpg/400px-Sitemap_google.jpg" width="400" height="635" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/20/Sitemap_google.jpg/600px-Sitemap_google.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/2/20/Sitemap_google.jpg/800px-Sitemap_google.jpg 2x" data-file-width="850" data-file-height="1350" /></a>  <div class="thumbcaption"><div class="magnify"><a href="/wiki/File:Sitemap_google.jpg" class="internal" title="Enlarge"></a></div>Sitemap of Google</div></div></div>
<p>Sitemaps may be addressed to users or to software. Many sites have user-visible sitemaps which present a systematic view, typically hierarchical, of the site. These are intended to help visitors find specific pages, and can also be used by crawlers. Alphabetically organized site maps, sometimes called site indexes, are a different approach.
</p><p>For use by search engines and other crawlers, there is a structured format, the XML Sitemap, which lists the pages in a site, their relative importance, and how often they are updated. This is pointed to from the <a href="/wiki/Robots_exclusion_standard" title="Robots exclusion standard">robots.txt file</a> and is typically called sitemap.xml. The structured format is particularly important for web sites which include pages that are not accessible through <a href="/wiki/Hyperlink" title="Hyperlink">links</a> from other pages, but only through the site's search tools or by dynamic construction of URLs in <a href="/wiki/Javascript" class="mw-redirect" title="Javascript">Javascript</a> or <a href="/wiki/Adobe_Flash" title="Adobe Flash">Adobe Flash</a>.
</p><p>They also act as a navigation aid <sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> by providing an overview of a site's content at a single glance.
</p>
<h2><span class="mw-headline" id="XML_sitemaps">XML sitemaps</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Site_map&amp;action=edit&amp;section=2" title="Edit section: XML sitemaps">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Sitemaps" title="Sitemaps">Sitemaps</a></div>
<p><a href="/wiki/Google" title="Google">Google</a> introduced <a href="/wiki/Sitemaps" title="Sitemaps">the Sitemaps protocol</a> so web developers can publish lists of links from across their sites. The basic premise is that some sites have a large number of dynamic pages that are only available through the use of forms and user entries. The Sitemap files contains URLs to these pages so that <a href="/wiki/Web_crawlers" class="mw-redirect" title="Web crawlers">web crawlers</a> can find them. <a href="/wiki/Bing.com" class="mw-redirect" title="Bing.com">Bing</a>, <a href="/wiki/Google" title="Google">Google</a>, <a href="/wiki/Yahoo" class="mw-redirect" title="Yahoo">Yahoo</a> and <a href="/wiki/Ask.com" title="Ask.com">Ask</a> now jointly support the Sitemaps protocol.
</p><p>Since the major <a href="/wiki/Search_engine" class="mw-redirect" title="Search engine">search engines</a> use the same protocol,<sup id="cite_ref-Oreilly_2-0" class="reference"><a href="#cite_note-Oreilly-2">&#91;2&#93;</a></sup> having a Sitemap lets them have the updated page information. Sitemaps do not guarantee all links will be crawled, and being crawled does not guarantee indexing.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> Google Webmaster Tools allow a website owner to upload a sitemap that Google will crawl, or they can accomplish the same thing with the robots.txt file.<sup id="cite_ref-Google_4-0" class="reference"><a href="#cite_note-Google-4">&#91;4&#93;</a></sup>
</p><p>XML Sitemaps have replaced the older method of "submitting to search engines" by filling out a form on the search engine's submission page. Now web developers submit a Sitemap directly, or wait for search engines to find it. Regularly submitting an updated sitemap when new pages are published may allow search engines to find and index those pages more quickly than it would by finding the pages on its own.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>
</p>

Sitemaps are a useful tool for making sites built in <a href="/wiki/Adobe_Flash" title="Adobe Flash">Flash</a> and other non-html languages searchable. If a website's navigation is built with Flash, an automated search program would probably only find the initial homepage; subsequent pages are unlikely to be found without an XML sitemap.

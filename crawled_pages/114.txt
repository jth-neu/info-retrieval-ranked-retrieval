<p><i><b>YaCy</b></i> (pronounced "ya see") is a <a href="/wiki/Free_software" title="Free software">free</a> <a href="/wiki/Distributed_search_engine" title="Distributed search engine">distributed search engine</a>, built on principles of <a href="/wiki/Peer-to-peer" title="Peer-to-peer">peer-to-peer</a> (P2P) networks.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> Its core is a <a href="/wiki/Computer_program" title="Computer program">computer program</a> written in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> distributed on several hundred computers, as of  September&#160;2006<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=YaCy&amp;action=edit">&#91;update&#93;</a></sup>, so-called YaCy-peers. Each YaCy-peer independently crawls through the Internet, analyzes and indexes found web pages, and stores indexing results in a common database (so called index) which is shared with other YaCy-peers using principles of P2P networks. It is a free <a href="/wiki/Web_search_engine" title="Web search engine">search engine</a> that everyone  can use to build a search portal for their intranet and to help search the public internet clearly.
</p><p>Compared to semi-distributed search engines, the YaCy-network has a decentralised architecture. All YaCy-peers are equal and no central <a href="/wiki/Server_(computing)" title="Server (computing)">server</a> exists. It can be run either in a <a href="/wiki/Web_crawler" title="Web crawler">crawling</a> mode or as a local <a href="/wiki/Proxy_server" title="Proxy server">proxy server</a>, indexing web pages visited by the person running YaCy on his or her computer. (Several mechanisms are provided to protect the user's privacy). Access to the search functions is made by a locally running web server which provides a search box to enter search terms, and returns search results in a similar format to other popular search engines.
</p><p>YaCy is available on <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>, <a href="/wiki/OS_X" class="mw-redirect" title="OS X">Mac</a> and <a href="/wiki/GNU/Linux" class="mw-redirect" title="GNU/Linux">GNU/Linux</a>. YaCy was created in 2003 by Michael Christen<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup>.
</p>

<h2><span class="mw-headline" id="System_components">System components</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=YaCy&amp;action=edit&amp;section=1" title="Edit section: System components">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>YaCy search engine is based on four elements:<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>
</p>
<dl><dt>Crawler</dt>
<dd>A search robot that traverses from web page to web page and analyzes their content.</dd>
<dt>Indexer</dt>
<dd>Creates a <a href="/wiki/Inverted_index" title="Inverted index">reverse word index</a> (RWI) i.e. each word from the RWI has its list of relevant URLs and ranking information. Words are saved in form of word hashes.</dd>
<dt>Search and administration interface</dt>
<dd>Made as a web interface provided by a local <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a> servlet with servlet engine.</dd>
<dt>Data storage</dt>
<dd>Used to store the reverse word index database utilizing a <a href="/wiki/Distributed_hash_table" title="Distributed hash table">distributed hash table</a>.</dd></dl>

<h2><span class="mw-headline" id="Search-engine_technology">Search-engine technology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=YaCy&amp;action=edit&amp;section=2" title="Edit section: Search-engine technology">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright"><div class="thumbinner" style="width:275px;"><a href="/wiki/File:YaCy_Network_Freeworld_01-12-2011.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/76/YaCy_Network_Freeworld_01-12-2011.png/273px-YaCy_Network_Freeworld_01-12-2011.png" width="273" height="154" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/7/76/YaCy_Network_Freeworld_01-12-2011.png/410px-YaCy_Network_Freeworld_01-12-2011.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/7/76/YaCy_Network_Freeworld_01-12-2011.png/546px-YaCy_Network_Freeworld_01-12-2011.png 2x" data-file-width="1024" data-file-height="576" /></a>  <div class="thumbcaption"><div class="magnify"><a href="/wiki/File:YaCy_Network_Freeworld_01-12-2011.png" class="internal" title="Enlarge"></a></div>YaCy network</div></div></div>
<ul><li><i>YaCy is a complete search <a href="/wiki/Software_appliance" title="Software appliance">appliance</a> with user interface, index, administration and monitoring.</i></li>
<li>YaCy harvests web pages with a web crawler. Documents are then parsed, indexed and the search index is stored locally. If your peer is part a peer network, then your local search index is also merged into the shared index for that network.</li>
<li>A search is started then the local index contributes together with a global search index from peers in the YaCy search <a href="/wiki/Network_protocol_design_principles" class="mw-redirect" title="Network protocol design principles">network</a>.</li></ul>

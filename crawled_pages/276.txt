<h1 id="firstHeading" class="firstHeading" lang="en">Web query classification</h1>			<div id="bodyContent" class="mw-body-content">

<p><b> </b>
A Web query topic classification/categorization is a problem in <a href="/wiki/Information_science" title="Information science">information science</a>. The task is to assign a <a href="/wiki/Web_search_query" title="Web search query">Web search query</a> to one or more predefined <a href="/wiki/Categorization" title="Categorization">categories</a>, based on its topics. The importance of query classification is underscored by many services provided by Web search. A direct application is to provide better search result pages for users with interests of different categories. For example, the users issuing a Web query “<i>apple</i>” might expect to see Web pages related to the fruit apple, or they may prefer to see products or news related to the computer company. Online advertisement services can rely on the query classification results to promote different products more accurately. Search result pages can be grouped according to the categories predicted by a query classification algorithm. However, the computation of query classification is non-trivial. Different from the <a href="/wiki/Document_classification" title="Document classification">document classification</a> tasks, queries submitted by Web search users are usually short and ambiguous; also the meanings of the queries are evolving over time. Therefore, query topic classification is much more difficult than traditional document classification tasks.
</p>

<h2><span class="mw-headline" id="KDDCUP_2005">KDDCUP 2005</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_query_classification&amp;action=edit&amp;section=1" title="Edit section: KDDCUP 2005">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>KDDCUP 2005 competition<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> highlighted the interests in query classification. The objective of this competition is to classify 800,000 real user queries into 67 target categories. Each query can belong to more than one target category. As an example of a QC task, given the query “<i>apple</i>”, it should be classified into ranked categories: “<i>Computers \ Hardware</i>; <i>Living \ Food &amp; Cooking</i>”.

<h2><span class="mw-headline" id="Difficulties">Difficulties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_query_classification&amp;action=edit&amp;section=2" title="Edit section: Difficulties">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Web query topic classification is to automatically assign a query to some predefined categories. Different from the traditional document classification tasks, there are several major difficulties which hinder the progress of Web <a href="/wiki/Query_understanding" title="Query understanding">query understanding</a>:
</p>
<h3><span id="How_to_derive_an_appropriate_feature_representation_for_Web_queries.3F"></span><span class="mw-headline" id="How_to_derive_an_appropriate_feature_representation_for_Web_queries?">How to derive an appropriate feature representation for Web queries?</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_query_classification&amp;action=edit&amp;section=3" title="Edit section: How to derive an appropriate feature representation for Web queries?">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Many queries are short and query terms are noisy. As an example, in the KDDCUP 2005 dataset, queries containing 3 words are most frequent (22%). Furthermore, 79% queries have no more than 4 words. A user query often has multiple meanings. For example, "<i>apple</i>" can mean a kind of fruit or a computer company. "<i>Java</i>" can mean a programming language or an island in Indonesia. In the KDDCUP 2005 dataset, most of the queries contain more than one meaning. Therefore, only using the keywords of the query to set up a <a href="/wiki/Vector_space_model" title="Vector space model">vector space model</a> for classification is not appropriate.
</p>
<ul><li>Query-enrichment based methods<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> start by enriching user queries to a collection of text documents through <a href="/wiki/Search_engines" class="mw-redirect" title="Search engines">search engines</a>. Thus, each query is represented by a pseudo-document which consists of the snippets of top ranked result pages retrieved by search engine. Subsequently, the text documents are classified into the target categories using synonym based classifier or statistical classifiers, such as <a href="/wiki/Naive_Bayes" class="mw-redirect" title="Naive Bayes">Naive Bayes</a> (NB) and <a href="/wiki/Support_Vector_Machines" class="mw-redirect" title="Support Vector Machines">Support Vector Machines</a> (SVMs).</li></ul>
<h3><span id="How_to_adapt_the_changes_of_the_queries_and_categories_over_time.3F"></span><span class="mw-headline" id="How_to_adapt_the_changes_of_the_queries_and_categories_over_time?">How to adapt the changes of the queries and categories over time?</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_query_classification&amp;action=edit&amp;section=4" title="Edit section: How to adapt the changes of the queries and categories over time?">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The meanings of queries may also evolve over time. Therefore, the old labeled training queries may be out-of-data and useless soon. How to make the classifier adaptive over time becomes a big issue. For example, the word "<i>Barcelona</i>" has a new meaning of the new micro-processor of AMD, while it refers to a city or football club before 2007. The distribution of the meanings of this term is therefore a function of time on the Web.
</p>
<ul><li>Intermediate taxonomy based method<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> first builds a bridging classifier on an intermediate taxonomy, such as <a href="/wiki/Open_Directory_Project" class="mw-redirect" title="Open Directory Project">Open Directory Project</a> (ODP), in an offline mode. This classifier is then used in an online mode to map user queries to the target categories via the intermediate taxonomy. The advantage of this approach is that the bridging classifier needs to be trained only once and is adaptive for each new set of target categories and incoming queries.</li></ul>
<h3><span id="How_to_use_the_unlabeled_query_logs_to_help_with_query_classification.3F"></span><span class="mw-headline" id="How_to_use_the_unlabeled_query_logs_to_help_with_query_classification?">How to use the unlabeled query logs to help with query classification?</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_query_classification&amp;action=edit&amp;section=5" title="Edit section: How to use the unlabeled query logs to help with query classification?">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Since the manually labeled training data for query classification is expensive, how to use a very large web search engine query log as a source of unlabeled data to aid in automatic query classification becomes a hot issue. These logs record the Web users' behavior when they search for information via a search engine. Over the years, query logs have become a rich resource which contains Web users' knowledge about the World Wide Web.
</p>
<ul><li>Query clustering method<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup> tries to associate related queries by clustering “session data”, which contain multiple queries and click-through information from a single user interaction. They take into account terms from result documents that a set of queries has in common. The use of query keywords together with session data is shown to be the most effective method of performing query clustering.</li>
<li>Selectional preference based method<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> tries to exploit some <a href="/wiki/Association_rules" class="mw-redirect" title="Association rules">association rules</a> between the query terms to help with the query classification. Given the training data, they exploit several classification approaches including exact-match using labeled data, N-Gram match using labeled data and classifiers based on perception. They emphasize on an approach adapted from computational linguistics named selectional preferences. If x and y form a pair (x; y) and y belongs to category c, then all other pairs (x; z) headed by x belong to c. They use unlabeled query log data to mine these rules and validate the effectiveness of their approaches on some labeled queries.</li></ul>
<h2><span class="mw-headline" id="Applications">Applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_query_classification&amp;action=edit&amp;section=6" title="Edit section: Applications">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul><li><b><a href="/wiki/Metasearch" class="mw-redirect" title="Metasearch">Metasearch engines</a></b> send a user's query to multiple search engines and blend the top results from each into one overall list. The search engine can organize the large number of Web pages in the search results, according to the potential categories of the issued query, for the convenience of Web users' navigation.</li>
<li><b><a href="/wiki/Vertical_search" title="Vertical search">Vertical search</a></b>, compared to general search, focuses on specific domains and addresses the particular information needs of niche audiences and professions. Once the search engine can predict the category of information a Web user is looking for, it can select a certain vertical search engine automatically, without forcing the user to access the vertical search engine explicitly.</li>
<li><b><a href="/wiki/Online_advertising" title="Online advertising">Online advertising</a></b><sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup><sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> aims at providing interesting advertisements to Web users during their search activities. The search engine can provide relevant advertising to Web users according to their interests, so that the Web users can save time and effort in research while the advertisers can reduce their advertising costs.</li></ul>
<p>All these services rely on the understanding Web users' search intents through their Web queries.
</p>

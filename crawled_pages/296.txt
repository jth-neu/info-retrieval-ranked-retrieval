<h1 id="firstHeading" class="firstHeading" lang="en">Semantic search</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
				<a class="mw-jump-link" href="#p-search">Jump to search</a>
				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Semantic search</b> seeks to improve <a href="/wiki/Search_engine_technology" title="Search engine technology">search</a> accuracy by understanding the searcher's <a href="/wiki/Intention" title="Intention">intent</a> and the <a href="/wiki/Context_(language_use)" title="Context (language use)">contextual</a> meaning of terms as they appear in the searchable dataspace, whether on the <a href="/wiki/World_Wide_Web" title="World Wide Web">Web</a> or within a closed system, to generate more relevant results.   Semantic search systems consider various points including context of search, location, intent, variation of words, <a href="/wiki/Synonym" title="Synonym">synonyms</a>, generalized and specialized queries, <a href="/wiki/Concept" title="Concept">concept</a> matching and <a href="/wiki/Natural_language" title="Natural language">natural language</a> queries to provide relevant search results.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> Major web search engines like Google and Bing incorporate some elements of semantic search. In <a href="/wiki/Vertical_search" title="Vertical search">vertical search</a>, <a href="/wiki/LinkedIn" title="LinkedIn">LinkedIn</a> publishes their semantic search approach to job search by recognizing and standardizing <a href="/wiki/Entity" title="Entity">entities</a> in both queries and documents, e.g., <a href="/wiki/Company" title="Company">companies</a>, <a href="/wiki/List_of_corporate_titles" title="List of corporate titles">titles</a> and <a href="/wiki/Skill" title="Skill">skills</a>, then constructing various entity-awared features based on the entities.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup>
</p><p>Guha et al. distinguish two major forms of search: navigational and research.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> In navigational search, the user is using the search engine as a navigation tool to navigate to a particular intended document. Semantic search is not applicable to navigational searches. In research search, the user provides the search engine with a phrase which is intended to denote an object about which the user is trying to gather/research information. There is no particular document which the user knows about and is trying to get to. Rather, the user is trying to locate a number of documents which together will provide the desired information. Semantic search lends itself well with this approach that is closely related with <a href="/wiki/Exploratory_search" title="Exploratory search">exploratory search</a>.
</p><p>Rather than using ranking algorithms such as Google's PageRank to predict relevancy, semantic search uses <a href="/wiki/Semantics" title="Semantics">semantics</a>, or the science of meaning in language, to produce highly relevant search results. In most cases, the goal is to deliver the information queried by a user rather than have a user sort through a list of loosely related keyword results. However, in 2012 Google also announced its own Semantic Search project.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>
</p><p>Author Seth Grimes lists "11 approaches that join <a href="/wiki/Semantics" title="Semantics">semantics</a> to search", and Hildebrand et al. provide an overview that lists semantic search systems and identifies other uses of semantics in the search process.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>
</p><p>Other authors primarily regard semantic search as a set of techniques for retrieving knowledge from richly structured data sources like <a href="/wiki/Ontology_(information_science)" title="Ontology (information science)">ontologies</a> and <a href="/wiki/XML" title="XML">XML</a> as found on the <a href="/wiki/Semantic_Web" title="Semantic Web">Semantic Web</a>.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> Such technologies enable the formal articulation of <a href="/wiki/Domain_knowledge" title="Domain knowledge">domain knowledge</a> at a high level of expressiveness and could enable the user to specify their intent in more detail at query time.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup>
</p>

<h2><span class="mw-headline" id="Disambiguation">Disambiguation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Semantic_search&amp;action=edit&amp;section=1" title="Edit section: Disambiguation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In order to understand what a user is searching for, <a href="/wiki/Word_sense_disambiguation" class="mw-redirect" title="Word sense disambiguation">word sense disambiguation</a> must occur. When a term is ambiguous, meaning it can have several meanings (for example, if one considers the <a href="/wiki/Lemma_(morphology)" title="Lemma (morphology)">lemma</a> "<a href="/wiki/Bark_(disambiguation)" class="mw-redirect mw-disambig" title="Bark (disambiguation)">bark</a>", which can be understood as "<a href="/wiki/Bark_(sound)" title="Bark (sound)">the sound of a dog</a>," "<a href="/wiki/Bark_(botany)" title="Bark (botany)">the skin of a tree</a>," or "<a href="/wiki/Barque" title="Barque">a three-masted sailing vessel</a>"), the disambiguation process is started, thanks to which the most probable meaning is chosen from all those possible.
</p><p>Such processes make use of other information present in a semantic analysis system and takes into account the meanings of other words present in the sentence and in the rest of the text. The determination of every meaning, in substance, influences the disambiguation of the others, until a situation of maximum <a href="/wiki/Plausibility" class="mw-redirect" title="Plausibility">plausibility</a> and coherence is reached for the sentence. All the fundamental information for the disambiguation process, that is, all the knowledge used by the system, is represented in the form of a semantic network, organized on a conceptual basis.
</p><p>In a structure of this type, every lexical concept coincides therefore with a semantic network node and is linked to others by specific semantic relationships in a hierarchical and hereditary structure. In this way, each concept is enriched with the characteristics and meaning of the nearby nodes.
</p><p>Every node of the network (called <a href="/wiki/Synset" class="mw-redirect" title="Synset">Synset</a>) groups a set of synonyms which represent the same lexical concept (called Synsets) and can contain:
</p>
<ul><li>single lemmata ('seat', 'vacation'; 'work', 'quick'; 'quickly', 'more', etc.)</li>
<li><a href="/wiki/Compound_word" class="mw-redirect" title="Compound word">compounds</a> ('non-stop', 'abat-jour', 'policeman')</li>
<li><a href="/wiki/Collocation" title="Collocation">collocations</a> ('credit card', 'university degree', 'treasury stock', 'go forward', etc.)</li></ul>
<p>The semantic relationships (links), which identify the semantic relationships between the synsets, are the order principals for the organization of the semantic network concepts.
</p>
<h2><span class="mw-headline" id="Commonly_used_searching_methodologies">Commonly used searching methodologies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Semantic_search&amp;action=edit&amp;section=2" title="Edit section: Commonly used searching methodologies">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<li>Graph Patterns - used to formulate patterns for locating interesting connecting paths between resources. Also commonly used in <a href="/wiki/Data_visualization" title="Data visualization">data visualization</a>.</li>
<li>Logics - by using <a href="/wiki/Inference" title="Inference">inference</a> based on <a href="/wiki/Web_Ontology_Language" title="Web Ontology Language">OWL</a></li>

<h2><span class="mw-headline" id="Ten_defining_attributes">Ten defining attributes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Semantic_search&amp;action=edit&amp;section=3" title="Edit section: Ten defining attributes">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The attributes of <i>semantic</i> search (those qualities that make it distinct from non-semantic search) are not all necessarily advantages by definition. For example, some attributes may improve search accuracy because of an exhaustive reiterative process but by effect overconsume time and/or resources. Accordingly, these ten attributes are merely salient features although the underlying assumption is that under perfect conditions they are generally preferable.
</p>
<ol><li>Handling morphological variations</li>
<li>Handling synonyms with correct senses</li>
<li>Handling generalizations</li>
<li>Handling concept matching</li>
<li>Handling knowledge matching</li>
<li>Handling natural language queries and questions</li>
<li>Ability to point to uninterrupted paragraph and the most relevant sentence</li>
<li>Ability to Customize and Organic Progress</li>
<li>Ability to operate without relying on statistics, user behavior, and other artificial means</li>
<li>Ability to detect its own performance</li></ol>

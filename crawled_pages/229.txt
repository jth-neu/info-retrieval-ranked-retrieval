<h1 id="firstHeading" class="firstHeading" lang="en">Wide area information server</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
				<a class="mw-jump-link" href="#p-search">Jump to search</a>
				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Wide Area Information Server</b> (<b>WAIS</b>) is a <a href="/wiki/Client%E2%80%93server" class="mw-redirect" title="Client–server">client–server</a> text searching system that uses the <a href="/wiki/American_National_Standards_Institute" title="American National Standards Institute">ANSI</a> Standard <a href="/wiki/Z39.50" title="Z39.50">Z39.50</a> Information Retrieval Service Definition and <a href="/wiki/Protocol_(computing)" class="mw-redirect" title="Protocol (computing)">Protocol</a> Specifications for Library Applications" (Z39.50:1988)  to search index <a href="/wiki/Database" title="Database">databases</a> on remote computers. It was developed in the late 1980s as a project of <a href="/wiki/Thinking_Machines_Corporation" title="Thinking Machines Corporation">Thinking Machines</a>, <a href="/wiki/Apple_Computer" class="mw-redirect" title="Apple Computer">Apple Computer</a>, <a href="/wiki/Dow_Jones" class="mw-redirect" title="Dow Jones">Dow Jones</a>, and <a href="/wiki/KPMG" title="KPMG">KPMG</a> Peat Marwick. 
</p><p>WAIS did not adhere to either the standard or its <a href="/wiki/OSI_Model" class="mw-redirect" title="OSI Model">OSI framework</a> (adopting instead TCP/IP) but created a unique protocol inspired by Z39.50:1988.
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wide_area_information_server&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<table class="plainlinks metadata ambox ambox-content ambox-Unreferenced" role="presentation"><tbody><tr><td class="mbox-image"><div style="width:52px"><a href="/wiki/File:Question_book-new.svg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/50px-Question_book-new.svg.png" width="50" height="39" srcset="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/75px-Question_book-new.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/100px-Question_book-new.svg.png 2x" data-file-width="512" data-file-height="399" /></a></div></td><td class="mbox-text"><div class="mbox-text-span">This section <b>does not <a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources">cite</a> any <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">sources</a></b>.<span class="hide-when-compact"> Please help <a class="external text" href="//en.wikipedia.org/w/index.php?title=Wide_area_information_server&amp;action=edit">improve this section</a> by <a href="/wiki/Help:Introduction_to_referencing_with_Wiki_Markup/1" title="Help:Introduction to referencing with Wiki Markup/1">adding citations to reliable sources</a>. <p>The WAIS protocol and servers were promoted by <a href="/wiki/Thinking_Machines_Corporation" title="Thinking Machines Corporation">Thinking Machines Corporation</a> (TMC) of <a href="/wiki/Cambridge,_Massachusetts" title="Cambridge, Massachusetts">Cambridge, Massachusetts</a>. TMC-produced WAIS servers ran on their massively parallel CM-2 (<a href="/wiki/Connection_Machine" title="Connection Machine">Connection Machine</a>) and <a href="/wiki/SPARC" title="SPARC">SPARC</a>-based CM-5 MP <a href="/wiki/Supercomputer" title="Supercomputer">supercomputers</a>. WAIS clients were developed for various <a href="/wiki/Operating_system" title="Operating system">operating systems</a> and <a href="/wiki/Windowing_system" title="Windowing system">windowing systems</a> including <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>, <a href="/wiki/Mac_OS" class="mw-redirect" title="Mac OS">Macintosh</a>, <a href="/wiki/NeXT" title="NeXT">NeXT</a>, <a href="/wiki/X_Window_System" title="X Window System">X</a>, <a href="/wiki/GNU_Emacs" title="GNU Emacs">GNU Emacs</a>, and character terminals.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> TMC released a free <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open source</a> version of WAIS for <a href="/wiki/Unix" title="Unix">Unix</a> in 1991. 
</p><p>Inspired by the WAIS project on full-text databases and emerging <a href="/wiki/SGML" class="mw-redirect" title="SGML">SGML</a> projects, Z39.50 version 2 (Z39.50:1992) was released. Unlike its 1988 predecessor, it was a compatible superset of the international ISO 10162/10163 standard.
</p><p>With the advent of Z39.50:1992, the termination of support for free WAIS by Thinking Machines and the establishment of WAIS Inc as a commercial venture, the U.S. <a href="/wiki/National_Science_Foundation" title="National Science Foundation">National Science Foundation</a> funded the <a href="/wiki/Clearinghouse_for_Networked_Information_Discovery_and_Retrieval" title="Clearinghouse for Networked Information Discovery and Retrieval">Clearinghouse for Networked Information Discovery and Retrieval</a> (CNIDR) to promote Internet search and discovery systems, open source and standards.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup>  CNIDR created a new, free open-source WAIS. This was the first freeWAIS based on the wais-8-b5 codebase of TMC, with a wholly new software suite <a href="/w/index.php?title=Isite&amp;action=edit&amp;redlink=1" class="new" title="Isite (page does not exist)">Isite</a> based upon Z39.50:1992 using <a href="/wiki/Isearch" title="Isearch">Isearch</a> as its full-text <a href="/wiki/Search_engine_(computing)" title="Search engine (computing)">search engine</a>.
</p><p>Ulrich Pfeifer and Norbert Gövert of the computer science department of the <a href="/wiki/University_of_Dortmund" class="mw-redirect" title="University of Dortmund">University of Dortmund</a> extended the CNIDR freeWAIS code to become freeWAIS-<b>sf</b> with <b>s</b>tructured <b>f</b>ields as its main improvement. Ulrich Pfeifer rewrote freeWAIS-sf in <a href="/wiki/Perl" title="Perl">Perl</a>, becoming WAIT.
</p><p>Inspired by WAIS' "Directory of Servers", <a href="/w/index.php?title=Eliot_Christian&amp;action=edit&amp;redlink=1" class="new" title="Eliot Christian (page does not exist)">Eliot Christian</a> of <a href="/wiki/USGS" class="mw-redirect" title="USGS">USGS</a> envisioned GILS: <a href="/w/index.php?title=Government_Information_Locator_Service&amp;action=edit&amp;redlink=1" class="new" title="Government Information Locator Service (page does not exist)">Government Information Locator Service</a>. GILS (based upon Z39.50:1992 with some extensions) became a U.S. Federal mandate as part of the <a href="/wiki/Paperwork_Reduction_Act" title="Paperwork Reduction Act">Paperwork Reduction Act of 1995</a> (<a href="/wiki/Title_44_of_the_United_States_Code" title="Title 44 of the United States Code">44 U.S.C.</a>&#160;<a rel="nofollow" class="external text" href="https://www.law.cornell.edu/uscode/text/44/3511">§&#160;3511</a>).
</p>
<h2><span class="mw-headline" id="Directory_of_Servers">Directory of Servers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wide_area_information_server&amp;action=edit&amp;section=2" title="Edit section: Directory of Servers">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<p>Thinking Machines Corp provided a service called the Directory of Servers. It was a WAIS server like any other information source except containing information about the other WAIS servers on the Internet. A WAIS server with TMC WAIS code creates a special record containing <a href="/wiki/Metadata" title="Metadata">metadata</a> plus some common words describing its indexed content. The record is uploaded to the central server and indexed along with the records from other public servers. The directory can be searched to find servers that might have content relevant to a specific field of interest. This model of searching for (WAIS) servers to search became the model for <a href="/w/index.php?title=Government_Information_Locator_Service&amp;action=edit&amp;redlink=1" class="new" title="Government Information Locator Service (page does not exist)">GILS</a> and Peter Deutsch's <a href="/wiki/WHOIS%2B%2B" title="WHOIS++">WHOIS++</a> <a href="/wiki/Distributed_computing" title="Distributed computing">distributed</a> <a href="/wiki/White_pages" class="mw-redirect" title="White pages">white pages</a> <a href="/wiki/Directory_service" title="Directory service">directory</a>.
</p>
<h2><span class="mw-headline" id="People">People</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wide_area_information_server&amp;action=edit&amp;section=3" title="Edit section: People">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<p>Two of the developers of WAIS, <a href="/wiki/Brewster_Kahle" title="Brewster Kahle">Brewster Kahle</a> and Harry Morris, left Thinking Machines to found WAIS Inc in Menlo Park, California with <a href="/wiki/Bruce_Gilliat" title="Bruce Gilliat">Bruce Gilliat</a>. WAIS Inc. was originally a joint project between Apple Computer, Peat Markwick, Dow Jones, and Thinking Machines. In 1992, the presidential campaign of <a href="/wiki/Ross_Perot" title="Ross Perot">Ross Perot</a> used the WAIS product as a campaign wide information system, connecting the field offices to the national office. Later, <a href="/wiki/Perot_Systems" title="Perot Systems">Perot Systems</a> adopted WAIS to better access the information in its corporate databases. Other early clients were the <a href="/wiki/Environmental_Protection_Agency" class="mw-redirect" title="Environmental Protection Agency">Environmental Protection Agency</a>, <a href="/wiki/Library_of_Congress" title="Library of Congress">Library of Congress</a>, and the <a href="/wiki/United_States_Department_of_Energy" title="United States Department of Energy">Department of Energy</a> and later the <i><a href="/wiki/Wall_Street_Journal" class="mw-redirect" title="Wall Street Journal">Wall Street Journal</a></i> and <i><a href="/wiki/Encyclop%C3%A6dia_Britannica" title="Encyclopædia Britannica">Encyclopædia Britannica</a></i>.
</p><p>WAIS Inc was sold to <a href="/wiki/AOL" title="AOL">AOL</a> in May 1995 for $15 million.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> Following the sale, Margaret St. Pierre left WAIS Inc to start Blue Angel Technologies. Her WAIS variant formed the basis of MetaStar. <a href="/wiki/Georgios_Papadopoulos" title="Georgios Papadopoulos">Georgios Papadopoulos</a> left to found <a href="/wiki/Atypon" title="Atypon">Atypon</a>. François Schiettecatte left <a href="/wiki/Human_Genome_Project" title="Human Genome Project">Human Genome Project</a> at <a href="/wiki/Johns_Hopkins_Hospital" title="Johns Hopkins Hospital">Johns Hopkins Hospital</a> and started FS-Consult and developed his own variant of WAIS which eventually became ScienceServer, which was later sold to <a href="/wiki/Elsevier_Science" class="mw-redirect" title="Elsevier Science">Elsevier Science</a>. Kahle and Gilliat went on to found the <a href="/wiki/Internet_Archive" title="Internet Archive">Internet Archive</a> and <a href="/wiki/Alexa_Internet" title="Alexa Internet">Alexa Internet</a>.
</p>
<h2><span class="mw-headline" id="WAIS_and_Gopher">WAIS and Gopher</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wide_area_information_server&amp;action=edit&amp;section=4" title="Edit section: WAIS and Gopher">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Public WAIS is often used as a full-text <a href="/wiki/Search_engine_(computing)" title="Search engine (computing)">search engine</a> for individual <a href="/wiki/Internet_Gopher" class="mw-redirect" title="Internet Gopher">Internet Gopher</a> servers, supplementing the popular <a href="/wiki/Veronica_(search_engine)" title="Veronica (search engine)">Veronica</a> system which only searches the menu titles of Gopher sites. WAIS and Gopher share the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>'s client–server architecture and a certain amount of its functionality. The WAIS protocol is influenced largely by the z39.50 protocol designed for networking library catalogs. It allows a text-based search, and retrieval following a search. Gopher provides a free text search mechanism, but principally uses menus. A menu is a list of titles, from which the user may pick one. While <a href="/wiki/Gopher_Space" class="mw-redirect" title="Gopher Space">Gopher Space</a> is a web containing many loops, the menu system gives the user the impression of a tree.<sup id="cite_ref-ReferenceA_4-0" class="reference"><a href="#cite_note-ReferenceA-4">&#91;4&#93;</a></sup><sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>
</p><p>The Web's data model is similar to the gopher model, except that menus are generalized to hypertext documents. In both cases, simple file servers generate the menus or hypertext directly from the file structure of a server. The Web's hypertext model permits the author more freedom to communicate the options available to the reader, as it can include headings and various forms of list structure.<sup id="cite_ref-ReferenceA_4-1" class="reference"><a href="#cite_note-ReferenceA-4">&#91;4&#93;</a></sup>
</p>

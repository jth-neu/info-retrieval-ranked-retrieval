<h1 id="firstHeading" class="firstHeading" lang="en">Meta element</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"><span class="mw-redirectedfrom">  (Redirected from <a href="/w/index.php?title=Meta_tags&amp;redirect=no" class="mw-redirect" title="Meta tags">Meta tags</a>)</span></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
				<a class="mw-jump-link" href="#p-search">Jump to search</a>
				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b><a href="/wiki/Meta" title="Meta">Meta</a> elements</b> are tags used in <a href="/wiki/HTML_element" title="HTML element">HTML</a> and <a href="/wiki/XHTML" title="XHTML">XHTML</a> documents to provide structured <a href="/wiki/Metadata" title="Metadata">metadata</a> about a <a href="/wiki/Web_page" title="Web page">Web page</a>. 
They are part of a web page's <code>head</code> section. Multiple Meta elements with different <a href="/wiki/XML#Key_terminology" title="XML">attributes</a> can be used on the same page. Meta elements can be used to specify page description, keywords and any other <a href="/wiki/Metadata" title="Metadata">metadata</a> not provided through the other <code>head</code> elements and attributes.
</p><p>The meta element has two uses: either to emulate the use of an <a href="/wiki/HTTP_response_header_field" class="mw-redirect" title="HTTP response header field">HTTP response header field</a>, or to embed additional metadata within the HTML document.
</p><p>With HTML up to and including <a href="/wiki/HTML_4.01" class="mw-redirect" title="HTML 4.01">HTML 4.01</a> and XHTML, there were four valid attributes: <code>content</code>, <code>http-equiv</code>, <code>name</code> and <code>scheme</code>. Under <a href="/wiki/HTML_5" class="mw-redirect" title="HTML 5">HTML 5</a> there are now five valid attributes, <code>charset</code> having been added. <code>http-equiv</code> is used to emulate an HTTP header, and <code>name</code> to embed metadata. The value of the statement, in either case, is contained in the <code>content</code> attribute, which is the only required attribute unless <code>charset</code> is given. <code>charset</code> is used to indicate the character set of the document, and is available in HTML5.
</p><p>Such elements must be placed as tags in the <code>head</code> section of an <a href="/wiki/HTML" title="HTML">HTML</a> or <a href="/wiki/XHTML" title="XHTML">XHTML</a> document.
</p><p>The two distinct parts of the elements are:
</p>
<ul><li>Title tags</li>
<li>Meta description</li></ul>

<h2><span class="mw-headline" id="Examples_of_the_meta_element">Examples of the <code>meta</code> element</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=1" title="Edit section: Examples of the meta element">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><code>meta</code> elements can specify <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">HTTP</a> headers which should be sent before the actual content when the HTML page is served from the web server to the client. For example:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">charset</span><span class="o">=</span><span class="s">&quot;utf-8&quot;</span><span class="p">&gt;</span>
</pre></div>
<p>as an alternative to the response header <code>Content-Type:</code> to indicate the <a href="/wiki/Media_type" title="Media type">media type</a> and, more commonly needed, the <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a> <a href="/wiki/Character_encoding" title="Character encoding">character encoding</a>.
</p><p>Meta tags can be used to describe the contents of the page:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;description&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;The Federal Aviation Administration is an operating mode of the U.S. Department of Transportation.&quot;</span><span class="p">&gt;</span>
</pre></div>
<p>In this example, the <code>meta</code> element describes the contents of a web page.
</p>
<h2><span class="mw-headline" id="Meta_element_used_in_search_engine_optimization">Meta element used in search engine optimization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=2" title="Edit section: Meta element used in search engine optimization">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Meta elements provide information about the web page, which can be used by search engines to help categorize the page correctly.
</p><p>They have been the focus of a field of <a href="/wiki/Marketing" title="Marketing">marketing</a> research known as <a href="/wiki/Search_engine_optimization" title="Search engine optimization">search engine optimization</a> (SEO), where different methods are used to provide a user's website with a higher ranking on search engines. Prior to the rise of content-analysis by search engines in the mid-1990s (most notably Google), search engines were reliant on meta data to correctly classify a Web page and webmasters quickly learned the commercial significance of having the right meta element.  The search engine community is now divided as to the value of meta tags.  Some claim they have no value, others that they are central, while many simply conclude there is no clear answer but, since they do no harm, they use them just in case.  Google<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> states they do support the meta tags "content", "robots", "google", "google-site-verification", "content-type", "refresh" and "google-bot".
</p><p>Major search engine robots look at many factors when determining how to rank a page of which meta tags will only form a portion.  Furthermore, most search engines change their ranking rules frequently.  Google have stated they update their ranking rules every 48 hours.  Under such circumstances, a definitive understanding of the role of meta tags in SEO is unlikely.
</p>
<h3><span class="mw-headline" id="The_keywords_attribute">The <code>keywords</code> attribute</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=3" title="Edit section: The keywords attribute">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <code>keywords</code> attribute was popularized by <a href="/wiki/Web_search_engine" title="Web search engine">search engines</a> such as <a href="/wiki/Infoseek" title="Infoseek">Infoseek</a> and <a href="/wiki/AltaVista" title="AltaVista">AltaVista</a> in 1995, and its popularity quickly grew until it became one of the most commonly used <code>meta</code> elements.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup>
</p><p>No consensus exists whether or not the <code>keywords</code> attribute has any effect on ranking at any of the major search engines today. It is speculated<sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Manual_of_Style/Words_to_watch#Unsupported_attributions" title="Wikipedia:Manual of Style/Words to watch"><span title="The material near this tag may use weasel words or too-vague attribution. (November 2015)">by whom?</span></a></i>&#93;</sup> that it does if the keywords used in the <code>meta</code> can also be found in the page copy itself.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2013)">citation needed</span></a></i>&#93;</sup> With respect to <a href="/wiki/Google" title="Google">Google</a>, thirty-seven leaders in search engine optimization concluded in April 2007 that the relevance of having keywords in the <code>meta</code>-attribute <code>keywords</code> is little to none<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> and in September 2009 <a href="/wiki/Matt_Cutts" title="Matt Cutts">Matt Cutts</a> of Google announced that they were no longer taking keywords into account whatsoever.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>  However, both these articles suggest that <a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a> still makes use of the keywords meta tag in some of its rankings.  Yahoo! itself claims support for the keywords meta tag in conjunction with other factors for improving search rankings.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup> In October 2009 Search Engine Round Table announced that "Yahoo Drops The Meta Keywords Tag Also"<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> but later reported that the announcement made by Yahoo!'s Senior Director of Search was incorrect.<sup id="cite_ref-gotitwrong_7-0" class="reference"><a href="#cite_note-gotitwrong-7">&#91;7&#93;</a></sup> In the corrected statement Yahoo! Senior Director of Search states that "…What changed with Yahoo's ranking algorithms is that while we still index the meta keyword tag, the ranking importance given to meta keyword tags receives the lowest ranking signal in our system … it will actually have less effect than introducing those same words in the body of the document, or any other section."<sup id="cite_ref-gotitwrong_7-1" class="reference"><a href="#cite_note-gotitwrong-7">&#91;7&#93;</a></sup> In Sept 2012, Google<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> announced that they will consider Keyword Meta tag for news publishers. <a href="/wiki/Google" title="Google">Google</a> said that this may help worthy content to get noticed.  The syntax of the news meta keyword<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup> has subtle difference from custom keyword meta tag; it is denoted by "news_keywords", while the custom keyword meta tag is denoted by "keywords".
</p>
<h3><span class="mw-headline" id="The_Title_attribute">The Title attribute</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=4" title="Edit section: The Title attribute">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>According to <a href="/wiki/Moz_(marketing_software)" title="Moz (marketing software)">Moz</a>, "Title tags are the second most important on-page factor for SEO, after content".<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup> They convey to the search engines what a given page is all about. It used to be standard SEO practice to include the primary and the secondary keywords in the title for better ranking.
</p><p>However, according to a 2016 study<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup> by Backlinko, it is possible that Google's leaning towards <a href="/wiki/Semantic_search" title="Semantic search">semantic search</a> has reduced the importance of placing keywords in the title.
</p><p>Regardless, the title tags still hold importance in three different ways.
</p>
<ul><li>They are displayed as page title in search results (and influence user behavior with respect to clicking on particular results).</li>
<li>Web browsers display them in naming open tabs; since the title is visible on hover, this is especially useful when too many tabs are open and only the <a href="/wiki/Favicon" title="Favicon">favicon</a> for each page (if available) is visible.</li>
<li>As in search results, titles are visible when page links are posted on social media and this, too, conveys to the users what the link is about.</li></ul>
<h3><span class="mw-headline" id="The_description_attribute">The <code>description</code> attribute</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=5" title="Edit section: The description attribute">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Unlike the <code>keywords</code> attribute, the <code>description</code> attribute is supported by most major search engines, like <a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a> and <a href="/wiki/Bing_(search_engine)" title="Bing (search engine)">Bing</a>, while <a href="/wiki/Google" title="Google">Google</a> will fall back on this tag when information about the page itself is requested (e.g. using the <tt>related:</tt> query). The <code>description</code> attribute provides a concise explanation of a <a href="/wiki/Web_page" title="Web page">Web page</a>'s content. This allows the Web page authors to give a more meaningful description for listings than might be displayed if the search engine was unable to automatically create its own description based on the page content. The description is often, but not always, displayed on <a href="/wiki/Search_engine_results_page" title="Search engine results page">search engine results pages</a>, so it can affect click-through rates. While clicks for a result can be a positive sign of effective <code>title</code> and <code>description</code> writing, <a href="/wiki/Google" title="Google">Google</a> does not recognize this meta element as a ranking factor,<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">&#91;12&#93;</a></sup> so using target keyword phrases in that element will not help a site rank better. W3C doesn't specify the size of this description meta tag, but almost all search engines recommend it to be shorter than 155 characters of plain text.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2007)">citation needed</span></a></i>&#93;</sup>
</p>
<h3><span class="mw-headline" id="The_language_attribute">The <code>language</code> attribute</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=6" title="Edit section: The language attribute">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <code>language</code> attribute tells search engines what natural language the website is written in (e.g. English, Spanish or French), as opposed to the coding language (e.g. HTML). It is normally an <a href="/wiki/IETF_language_tag" title="IETF language tag">IETF language tag</a> for the language name. It is of most use when a website is written in multiple languages and can be included on each page to tell search engines in which language a particular page is written.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup>
</p>
<h3><span class="mw-headline" id="The_robots_attribute">The <code>robots</code> attribute</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=7" title="Edit section: The robots attribute">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <code>robots</code> attribute, supported by several major search engines,<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup><sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability"><span title="The material near this tag failed verification of its source citation(s). (January 2010)">not in citation given</span></a></i>&#93;</sup> controls whether search engine spiders are allowed to <a href="/wiki/Search_engine_indexing" title="Search engine indexing">index</a> a page, or not, and whether they should follow links from a page, or not. The attribute can contain one or more comma-separate values. The <code>noindex</code> value prevents a page from being indexed, and <code>nofollow</code> prevents links from being <a href="/wiki/Web_crawler" title="Web crawler">crawled</a>. Other values recognized by one or more search engines can influence how the engine indexes pages, and how those pages appear on the search results.  These include <code>noarchive</code>, which instructs a search engine not to store an archived copy of the page, and <code>nosnippet</code>, which asks that the search engine not include a snippet from the page along with the page's listing in search results.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup>
</p><p>Meta tags are one of the best options for preventing search engines from indexing content of a website.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup>
</p>
<h4><span class="mw-headline" id="Additional_attributes_for_search_engines">Additional attributes for search engines</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=8" title="Edit section: Additional attributes for search engines">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<h5><span class="mw-headline" id="NOODP">NOODP</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=9" title="Edit section: NOODP">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p>The search engines <a href="/wiki/Google" title="Google">Google</a>, <a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a> and <a href="/wiki/Live_Search" class="mw-redirect" title="Live Search">MSN</a> use in some cases the title and abstract of the <a href="/wiki/DMOZ" title="DMOZ">DMOZ</a> (aka Open Directory Project) listing of a website for the title and/or description (also called snippet or abstract) in the <a href="/wiki/Search_engine_results_page" title="Search engine results page">search engine results pages</a> (SERP). To give <a href="/wiki/Webmaster" title="Webmaster">webmasters</a> the option to specify that the Open Directory Project content should not be used for listings of their website, Microsoft introduced in May 2006 the new "<code>NOODP</code>" value for the "<code>robots</code>" element of the meta tags.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">&#91;17&#93;</a></sup> Google followed in July 2006<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup> and Yahoo! in October 2006.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup>
</p><p>The syntax is the same for all search engines who support the tag.
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;robots&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;noodp&quot;</span> <span class="p">&gt;</span>
</pre></div>
<p>Webmasters can decide if they want to disallow the use of their ODP listing on a per search engine basis
</p><p>Google:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;googlebot&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;noodp&quot;</span> <span class="p">&gt;</span>
</pre></div>
<p>Yahoo!
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;Slurp&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;noodp&quot;</span> <span class="p">&gt;</span>
</pre></div><p>
MSN and Live Search (via <a href="/wiki/Bingbot" title="Bingbot">bingbot</a>, previously <a href="/wiki/Msnbot" title="Msnbot">msnbot</a>): </p><div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;bingbot&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;noodp&quot;</span> <span class="p">&gt;</span>
</pre></div>
<h5><span class="mw-headline" id="NOYDIR">NOYDIR</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=10" title="Edit section: NOYDIR">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p>Yahoo! puts content from their own Yahoo! directory next to the ODP listing. In 2007 they introduced a meta tag that lets web designers opt out of this.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">&#91;20&#93;</a></sup>
</p><p>Adding the <code>NOYDIR</code> tag to a page will prevent Yahoo! from displaying <a href="/wiki/Yahoo!_Directory" title="Yahoo! Directory">Yahoo! Directory</a> titles and abstracts.
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;robots&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;noydir&quot;</span> <span class="p">&gt;</span>
<span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;Slurp&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;noydir&quot;</span> <span class="p">&gt;</span>
</pre></div>
<h5><span class="mw-headline" id="Static_HTML_from_Ajax_hash_fragment_URLs">Static HTML from Ajax hash fragment URLs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=11" title="Edit section: Static HTML from Ajax hash fragment URLs">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;fragment&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;!&quot;</span> <span class="p">&gt;</span>
</pre></div>
<p>For dynamically created web pages, Google proposes the above meta tag which causes fragment URLs (ones that look like "<a rel="nofollow" class="external free" href="http://www.url.com/#xyz">http://www.url.com/#xyz</a>" ) to be rewritten and recrawled as "ugly URLs" (i.e. ones looking like "<a rel="nofollow" class="external free" href="http://www.url.com/?_escaped_fragment_=xyz">http://www.url.com/?_escaped_fragment_=xyz</a>" ).  See <sup id="cite_ref-21" class="reference"><a href="#cite_note-21">&#91;21&#93;</a></sup> for more details about the rewriting process.  This rewrite step is a signal to the web site to please provide a simple and full HTML web page that is the result of executing the AJAX or other scripting on the page.  This allows Google and other search engines to collect and index static web pages even when a web page is dynamically created and updated by the browser.
</p>
<h5><span class="mw-headline" id="Robots-NoContent">Robots-NoContent</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=12" title="Edit section: Robots-NoContent">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p><a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a> also introduced in May 2007 the attribute value: <code>class="robots-nocontent"</code>.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">&#91;22&#93;</a></sup> This is not a meta tag, but an attribute and value, which can be used throughout Web page tags where needed. Content of the page where this attribute is being used will be ignored by the Yahoo! crawler and not included in the search engine's index.
</p><p>Examples for the use of the <code>robots-nocontent</code> tag:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">div</span> <span class="na">class</span><span class="o">=</span><span class="s">&quot;robots-nocontent&quot;</span><span class="p">&gt;</span>excluded content<span class="p">&lt;/</span><span class="nt">div</span><span class="p">&gt;</span>
<span class="p">&lt;</span><span class="nt">span</span> <span class="na">class</span><span class="o">=</span><span class="s">&quot;robots-nocontent&quot;</span><span class="p">&gt;</span>excluded content<span class="p">&lt;/</span><span class="nt">span</span><span class="p">&gt;</span>
<span class="p">&lt;</span><span class="nt">p</span> <span class="na">class</span><span class="o">=</span><span class="s">&quot;robots-nocontent&quot;</span><span class="p">&gt;</span>excluded content<span class="p">&lt;/</span><span class="nt">p</span><span class="p">&gt;</span>
</pre></div>
<h3><span class="mw-headline" id="Academic_studies">Academic studies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=13" title="Edit section: Academic studies">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Google" title="Google">Google</a> does not use HTML keyword or meta tag elements for indexing. The Director of Research at Google, <a href="/wiki/Monika_Henzinger" title="Monika Henzinger">Monika Henzinger</a>, was quoted (in 2002) as saying, "Currently we don't trust metadata because we are afraid of being manipulated." <sup id="cite_ref-23" class="reference"><a href="#cite_note-23">&#91;23&#93;</a></sup> Other search engines developed techniques to penalize Web sites considered to be "cheating the system". For example, a Web site repeating the same meta keyword several times may have its ranking <i>decreased</i> by a search engine trying to eliminate this practice, though that is unlikely. It is more likely that a search engine will ignore the meta keyword element completely, and most do regardless of how many words are used in the element.
</p><p><a href="/wiki/Google" title="Google">Google</a> does, however, use meta tag elements for displaying site links. The title tags are used to create the link in search results:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">title</span><span class="p">&gt;</span>Site name - Page title - Keyword description<span class="p">&lt;/</span><span class="nt">title</span><span class="p">&gt;</span>
</pre></div>
<p>The meta description often appears in <a href="/wiki/Google" title="Google">Google</a> search results to describe the link:
</p>
<div class="mw-highlight mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;description&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;A blurb to describe the content of the page appears here&quot;</span> <span class="p">&gt;</span>
</pre></div><p>Additionally, enterprise search startup <a href="/wiki/Swiftype" title="Swiftype">Swiftype</a> considers meta tags as a mechanism for signaling relevancy for their web site search engines, even introducing their own extension called Meta Tags 2.<sup id="cite_ref-24" class="reference"><a href="#cite_note-24">&#91;24&#93;</a></sup>
</p><h2><span class="mw-headline" id="Redirects">Redirects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=14" title="Edit section: Redirects">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Meta_refresh" title="Meta refresh">Meta refresh</a> elements can be used to instruct a <a href="/wiki/Web_browser" title="Web browser">Web browser</a> to automatically refresh a Web page after a given time interval. It is also possible to specify an alternative <a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">URL</a> and use this technique in order to <a href="/wiki/URL_redirection" title="URL redirection">redirect</a> the user to a different location.
Auto refreshing via a META element has been deprecated for more than ten years,<sup id="cite_ref-25" class="reference"><a href="#cite_note-25">&#91;25&#93;</a></sup> and recognized as problematic before that.<sup id="cite_ref-26" class="reference"><a href="#cite_note-26">&#91;26&#93;</a></sup>
</p><p>The W3C suggests that user agents should allow users to disable it, otherwise META refresh should not be used by web pages.
For Internet Explorer's security settings, under the miscellaneous category, meta refresh can be turned off by the user, thereby disabling its redirect ability.
In Mozilla Firefox it can be disabled in the configuration file under the key name "accessibility.blockautorefresh".<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">&#91;27&#93;</a></sup>
</p><p>Many web design tutorials also point out that client-side redirecting tends to interfere with the normal functioning of a Web browser's "back" button. After being redirected, clicking the back button will cause the user to go back to the redirect page, which redirects them again. Some modern browsers seem to overcome this problem however, including <a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a>, <a href="/wiki/Mozilla_Firefox" class="mw-redirect" title="Mozilla Firefox">Mozilla Firefox</a> and <a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2011)">citation needed</span></a></i>&#93;</sup>
</p><p>Auto-redirects via markup (versus server-side redirects) are not in compliance with the <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">W3C's</a> - <a href="/wiki/Web_Content_Accessibility_Guidelines" title="Web Content Accessibility Guidelines">Web Content Accessibility Guidelines (WCAG) 1.0</a> (guideline 7.5).<sup id="cite_ref-WCAG_28-0" class="reference"><a href="#cite_note-WCAG-28">&#91;28&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="HTTP_message_headers">HTTP message headers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=15" title="Edit section: HTTP message headers">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Meta elements of the form <code>&lt;meta http-equiv="foo" content="bar"&gt;</code> can be used as alternatives to HTTP headers. For example, <code>&lt;meta http-equiv="expires" content="Wed, 21 June 2006 14:25:27 GMT"&gt;</code> would tell the browser that the page "expires" on June 21, 2006 at 14:25:27 GMT and that it may safely cache the page until then. The <a href="/wiki/HTML" title="HTML">HTML 4.01</a> specification optionally allows this tag to be parsed by <a href="/wiki/HTTP_server" class="mw-redirect" title="HTTP server">HTTP servers</a> and set as part of the HTTP response headers,<sup id="cite_ref-HTML_4.01_specification_29-0" class="reference"><a href="#cite_note-HTML_4.01_specification-29">&#91;29&#93;</a></sup> but no web servers currently implement this behavior.<sup id="cite_ref-http-equiv_HTML5_30-0" class="reference"><a href="#cite_note-http-equiv_HTML5-30">&#91;30&#93;</a></sup> Instead, the user agent emulates the behavior for some HTTP headers as if they had been sent in the response header itself.
</p>
<h2><span class="mw-headline" id="Alternative_to_meta_elements">Alternative to <code>meta</code> elements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta_element&amp;action=edit&amp;section=16" title="Edit section: Alternative to meta elements">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Some HTML elements and attributes already handle certain pieces of meta data and may be used by authors instead of META to specify those pieces: the TITLE element, the ADDRESS element, the INS and DEL elements, the title attribute, and the cite attribute.<sup id="cite_ref-w3.org_31-0" class="reference"><a href="#cite_note-w3.org-31">&#91;31&#93;</a></sup>
</p><p>An alternative to <code>meta</code> elements for enhanced subject access within a website is the use of a back-of-book-style index for the website. See the <a rel="nofollow" class="external text" href="http://www.asindexing.org">American Society of Indexers</a> website for an example.
</p><p>In 1994, <a href="/wiki/Aliweb" title="Aliweb">ALIWEB</a>, also used an index file to provide the type of information commonly found in meta keywords attributes.
</p><p>In cases where the content attribute's value is a URL, many authors decide to use a link element with a proper value for its rel attribute as well.<sup id="cite_ref-w3.org_31-1" class="reference"><a href="#cite_note-w3.org-31">&#91;31&#93;</a></sup>
</p><p>For a comparison on when it is best to use HTTP-headers, meta-elements, or attributes in the case of language specification: <a rel="nofollow" class="external text" href="http://www.w3.org/International/questions/qa-http-and-lang">see here</a>.
</p>

<h1 id="firstHeading" class="firstHeading" lang="en">Document retrieval</h1>			<div id="bodyContent" class="mw-body-content">
				<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>				<div id="contentSub"></div>
				<div id="jump-to-nav"></div>				<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
				<a class="mw-jump-link" href="#p-search">Jump to search</a>
				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Document retrieval</b> is defined as the matching of some stated user query against a set of <a href="/wiki/Free-text" class="mw-redirect" title="Free-text">free-text</a> records. These records could be any type of mainly <a href="/wiki/Natural_language" title="Natural language">unstructured text</a>, such as <a href="/wiki/Newspaper_article" class="mw-redirect" title="Newspaper article">newspaper articles</a>, real estate records or paragraphs in a manual. User queries can range from multi-sentence full descriptions of an information need to a few words.
</p><p>Document retrieval is sometimes referred to as, or as a branch of, <b>text retrieval</b>. Text retrieval is a branch of <a href="/wiki/Information_retrieval" title="Information retrieval">information retrieval</a> where the information is stored primarily in the form of <a href="/wiki/Natural_language" title="Natural language">text</a>. Text databases became decentralized thanks to the <a href="/wiki/Personal_computer" title="Personal computer">personal computer</a> and the <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROM</a>. Text retrieval is a critical area of study today, since it is the fundamental basis of all <a href="/wiki/Internet" title="Internet">internet</a> <a href="/wiki/Search_engine" class="mw-redirect" title="Search engine">search engines</a>.
</p>

<h2><span class="mw-headline" id="Description">Description</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Document_retrieval&amp;action=edit&amp;section=1" title="Edit section: Description">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Document retrieval systems find information to given criteria by matching text records (<i>documents</i>) against user queries, as opposed to <a href="/wiki/Expert_system" title="Expert system">expert systems</a> that answer questions by <a href="/wiki/Inference" title="Inference">inferring</a> over a logical <a href="/wiki/Knowledge_base" title="Knowledge base">knowledge database</a>. A document retrieval system consists of a database of documents, a <a href="/wiki/Classification_algorithm" class="mw-redirect" title="Classification algorithm">classification algorithm</a> to build a full text index, and a user interface to access the database.
</p><p>A document retrieval system has two main tasks:
</p>
<ol><li>Find relevant documents to user queries</li>
<li>Evaluate the matching results and sort them according to relevance, using algorithms such as <a href="/wiki/PageRank" title="PageRank">PageRank</a>.</li></ol>
<p>Internet <a href="/wiki/Search_engines" class="mw-redirect" title="Search engines">search engines</a> are classical applications of document retrieval. The vast majority of retrieval systems currently in use range from simple Boolean systems through to systems using <a href="/wiki/Statistical" class="mw-redirect" title="Statistical">statistical</a> or <a href="/wiki/Natural_language_processing" title="Natural language processing">natural language processing</a> techniques.
</p>
<h2><span class="mw-headline" id="Variations">Variations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Document_retrieval&amp;action=edit&amp;section=2" title="Edit section: Variations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are two main classes of indexing schemata for document retrieval systems: <i>form based</i> (or <i>word based</i>), and <i>content based</i> indexing. The document classification scheme (or <a href="/wiki/Search_engine_indexing" title="Search engine indexing">indexing algorithm</a>) in use determines the nature of the document retrieval system.
</p>
<h3><span class="mw-headline" id="Form_based">Form based</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Document_retrieval&amp;action=edit&amp;section=3" title="Edit section: Form based">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Form based document retrieval addresses the exact syntactic properties of a text, comparable to substring matching in string searches. The text is generally unstructured and not necessarily in a natural language, the system could for example be used to process large sets of chemical representations in molecular biology. A <a href="/wiki/Suffix_tree" title="Suffix tree">suffix tree</a> algorithm is an example for form based indexing.
</p>
<h3><span class="mw-headline" id="Content_based">Content based</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Document_retrieval&amp;action=edit&amp;section=4" title="Edit section: Content based">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The content based approach exploits semantic connections between documents and parts thereof, and semantic connections between queries and documents. Most content based document retrieval systems use an <a href="/wiki/Inverted_index" title="Inverted index">inverted index</a> algorithm.
</p><p>A <i>signature file</i> is a technique that creates a <i>quick and dirty</i> filter, for example a <a href="/wiki/Bloom_filter" title="Bloom filter">Bloom filter</a>, that will keep all the documents that match to the query and <i>hopefully</i> a few ones that do not. The way this is done is by creating for each file a signature, typically a hash coded version. One method is superimposed coding. A post-processing step is done to discard the false alarms. Since in most cases this structure is inferior to <a href="/wiki/Inverted_file" class="mw-redirect" title="Inverted file">inverted files</a> in terms of speed, size and functionality, it is not used widely. However, with proper parameters it can beat the inverted files in certain environments.
</p>

<p>The <b>Gopher</b> protocol <span class="nowrap"><span class="IPA nopopups noexcerpt"><a href="/wiki/Help:IPA/English" title="Help:IPA/English">/<span style="border-bottom:1px dotted"><span title="/ˈ/: primary stress follows">ˈ</span><span title="/ɡ/: &#39;g&#39; in &#39;guy&#39;">ɡ</span><span title="/oʊ/: &#39;o&#39; in &#39;code&#39;">oʊ</span><span title="&#39;f&#39; in &#39;find&#39;">f</span><span title="/ər/: &#39;er&#39; in &#39;letter&#39;">ər</span></span>/</a></span></span> is a <a href="/wiki/Internet_protocol_suite" title="Internet protocol suite">TCP/IP</a> <a href="/wiki/Application_layer" title="Application layer">application layer</a> <a href="/wiki/Communications_protocol" class="mw-redirect" title="Communications protocol">protocol</a> designed for distributing, searching, and retrieving documents over the Internet. The Gopher protocol was strongly oriented towards a menu-document design and presented an alternative to the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> in <a href="/wiki/History_of_the_World_Wide_Web" title="History of the World Wide Web">its early stages</a>, but ultimately <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">Hypertext Transfer Protocol</a> (HTTP) became the dominant protocol. The Gopher ecosystem is often regarded as the effective predecessor of the World Wide Web.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup>
</p><p>The protocol was invented by a team led by <a href="/wiki/Mark_P._McCahill" title="Mark P. McCahill">Mark P. McCahill</a><sup id="cite_ref-tri_2-0" class="reference"><a href="#cite_note-tri-2">&#91;2&#93;</a></sup> at the <a href="/wiki/University_of_Minnesota" title="University of Minnesota">University of Minnesota</a>.  It offers some features not natively supported by the Web and imposes a much stronger hierarchy on information stored on it. Its text menu interface is well-suited to computing environments that rely heavily on remote <a href="/wiki/Computer_terminal#Text_terminals" title="Computer terminal">text-oriented computer terminals</a>, which were still common at the time of its creation in 1991, and the simplicity of its protocol facilitated a wide variety of client implementations. More recent Gopher revisions and graphical clients added support for multimedia.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>&#93;</sup> Gopher was preferred by many network administrators for using fewer network resources than Web services.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup>
</p><p>Gopher's hierarchical structure provided a platform for the first large-scale electronic library connections.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> Gopher has been described by some enthusiasts<sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Manual_of_Style/Words_to_watch#Unsupported_attributions" title="Wikipedia:Manual of Style/Words to watch"><span title="The material near this tag possibly uses too-vague attribution or weasel words. (September 2018)">who?</span></a></i>&#93;</sup> as "faster and more efficient and so much more organised" than today's Web services. The Gopher protocol is still in use by enthusiasts, and although it has been almost entirely supplanted by the Web, a small population of actively-maintained servers remains.
</p>

<h2><span class="mw-headline" id="Origins">Origins</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=1" title="Edit section: Origins">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Gopher system was released in mid-1991 by Mark P. McCahill, <a href="/w/index.php?title=Farhad_Anklesaria&amp;action=edit&amp;redlink=1" class="new" title="Farhad Anklesaria (page does not exist)">Farhad Anklesaria</a>, Paul Lindner, <a href="/w/index.php?title=Daniel_Torrey&amp;action=edit&amp;redlink=1" class="new" title="Daniel Torrey (page does not exist)">Daniel Torrey</a>, and Bob Alberti of the <a href="/wiki/University_of_Minnesota" title="University of Minnesota">University of Minnesota</a><sup id="cite_ref-wwwunleashed_5-0" class="reference"><a href="#cite_note-wwwunleashed-5">&#91;5&#93;</a></sup> in the United States. Its central goals were, as stated in <a class="external mw-magiclink-rfc" rel="nofollow" href="https://tools.ietf.org/html/rfc1436">RFC 1436</a>:
</p>
<ul><li>A file-like hierarchical arrangement that would be familiar to users.</li>
<li>A simple syntax.</li>
<li>A system that can be created quickly and inexpensively.</li>
<li>Extending the file system metaphor, such as searches.</li></ul>
<p>Gopher combines document hierarchies with collections of services, including <a href="/wiki/Wide_area_information_server" title="Wide area information server">WAIS</a>, the <a href="/wiki/Archie_search_engine" title="Archie search engine">Archie</a> and <a href="/wiki/Veronica_(search_engine)" title="Veronica (search engine)">Veronica</a> <a href="/wiki/Search_engine_(computing)" title="Search engine (computing)">search engines</a>, and gateways to other information systems such as <a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">File Transfer Protocol</a> (FTP) and <a href="/wiki/Usenet" title="Usenet">Usenet</a>.
</p><p>The general interest in <a href="/w/index.php?title=Campus-Wide_Information_Systems&amp;action=edit&amp;redlink=1" class="new" title="Campus-Wide Information Systems (page does not exist)">Campus-Wide Information Systems</a> (CWISs)<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> in higher education at the time, and the ease with which a Gopher server could be set up to create an instant CWIS with links to other sites' online directories and resources were the factors contributing to Gopher's rapid adoption.
</p><p>The name was coined by Anklesaria as a play on several meanings of the word "gopher".<sup id="cite_ref-gophervideo_7-0" class="reference"><a href="#cite_note-gophervideo-7">&#91;7&#93;</a></sup> The <a href="/wiki/University_of_Minnesota" title="University of Minnesota">University of Minnesota</a> mascot is the gopher,<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> a <a href="/wiki/Gofer" title="Gofer">gofer</a> is an assistant who "goes for" things, and a <a href="/wiki/Gopher" title="Gopher">gopher</a> burrows through the ground to reach a desired location.<sup id="cite_ref-gopher_protocol_rise_and_fall_9-0" class="reference"><a href="#cite_note-gopher_protocol_rise_and_fall-9">&#91;9&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Decline">Decline</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=2" title="Edit section: Decline">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> was in its infancy in 1991, and Gopher services quickly became established. By the late 1990s, Gopher had ceased expanding. Several factors contributed to Gopher's stagnation:
</p>
<ul><li>In February 1993, the <a href="/wiki/University_of_Minnesota" title="University of Minnesota">University of Minnesota</a> announced that it would charge licensing fees for the use of its implementation of the Gopher server.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup><sup id="cite_ref-gopher_protocol_rise_and_fall_9-1" class="reference"><a href="#cite_note-gopher_protocol_rise_and_fall-9">&#91;9&#93;</a></sup> Users became concerned that fees might also be charged for independent implementations.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup><sup id="cite_ref-12" class="reference"><a href="#cite_note-12">&#91;12&#93;</a></sup>  Gopher expansion stagnated, to the advantage of the World Wide Web, to which <a href="/wiki/CERN" title="CERN">CERN</a> disclaimed ownership.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup> In September 2000, the University of Minnesota re-licensed its Gopher software under the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a>.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup></li>
<li>Gopher client functionality was quickly duplicated by the early <a href="/wiki/Mosaic_(web_browser)" title="Mosaic (web browser)">Mosaic</a> web browser, which subsumed its protocol.</li>
<li>Gopher has a more rigid structure than the free-form <a href="/wiki/HTML" title="HTML">HTML</a> of the Web. Every Gopher document has a defined format and type, and the typical user navigates through a single server-defined menu system to get to a particular document. This can be quite different from the way a user finds documents on the Web.</li></ul>
<p>Gopher remains in active use by its enthusiasts, and there have been attempts to revive Gopher on modern platforms and mobile devices. One attempt is The Overbite Project, which hosts various browser extensions and modern clients.
</p>
<h3><span class="mw-headline" id="Server_census">Server census</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=3" title="Edit section: Server census">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul><li>As of  2012<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Gopher_(protocol)&amp;action=edit">&#91;update&#93;</a></sup>, there remained about 160 gopher servers indexed by <a href="/wiki/Veronica_(search_engine)" title="Veronica (search engine)">Veronica-2</a>,<sup id="cite_ref-Veronica_report_15-0" class="reference"><a href="#cite_note-Veronica_report-15">&#91;15&#93;</a></sup> reflecting a slow growth from 2007 when there were fewer than 100.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup> They are typically infrequently updated. On these servers Veronica indexed approximately 2.5 million unique selectors. A handful of new servers were being set up every year by hobbyists with over 50 having been set up and added to Floodgap's list since 1999.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">&#91;17&#93;</a></sup> A snapshot of Gopherspace in 2007 circulated on <a href="/wiki/BitTorrent" title="BitTorrent">BitTorrent</a> and was still available in 2010.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup> Due to the simplicity of the Gopher protocol, setting up new servers or adding Gopher support to browsers is often done in a <a href="/wiki/Tongue-in-cheek" title="Tongue-in-cheek">tongue-in-cheek</a> manner, principally on <a href="/wiki/April_Fools%27_Day" title="April Fools&#39; Day">April Fools' Day</a>.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup></li></ul>
<ul><li>In November 2014 Veronica indexed 144 gopher servers,<sup id="cite_ref-Veronica_report_15-1" class="reference"><a href="#cite_note-Veronica_report-15">&#91;15&#93;</a></sup> reflecting a small drop from 2012, but within these servers Veronica indexed approximately 3&#160;million unique selectors.</li></ul>
<ul><li>In March 2016 Veronica indexed 135 gopher servers,<sup id="cite_ref-Veronica_report_15-2" class="reference"><a href="#cite_note-Veronica_report-15">&#91;15&#93;</a></sup> within which it indexed approximately 4&#160;million unique selectors.</li></ul>
<ul><li>In March 2017 Veronica indexed 133 gopher servers,<sup id="cite_ref-Veronica_report_15-3" class="reference"><a href="#cite_note-Veronica_report-15">&#91;15&#93;</a></sup> within which it indexed approximately 4.9&#160;million unique selectors.</li></ul>
<ul><li>In May 2018 Veronica indexed 260 gopher servers, <sup id="cite_ref-Veronica_report_15-4" class="reference"><a href="#cite_note-Veronica_report-15">&#91;15&#93;</a></sup> within which it indexed approximately 3.7&#160;million unique selectors.</li></ul>
<h2><span class="mw-headline" id="Technical_details">Technical details</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=4" title="Edit section: Technical details">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The conceptualization of knowledge in "Gopher space" or a "cloud" as specific information in a particular file, and the prominence of the FTP, influenced the technology and the resulting functionality of Gopher.
</p>
<h3><span class="mw-headline" id="Gopher_characteristics">Gopher characteristics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=5" title="Edit section: Gopher characteristics">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Gopher is designed to function and to appear much like a mountable read-only global <a href="/wiki/Distributed_file_system" class="mw-redirect" title="Distributed file system">network file system</a> (and software, such as <a href="/w/index.php?title=Gopherfs&amp;action=edit&amp;redlink=1" class="new" title="Gopherfs (page does not exist)">gopherfs</a>, is available that can actually mount a Gopher server as a <a href="/wiki/Filesystem_in_Userspace" title="Filesystem in Userspace">FUSE</a> resource).  At a minimum, whatever a person can do with data files on a <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROM</a>, one can do on Gopher.
</p><p>A Gopher system consists of a series of hierarchical hyperlinkable menus. The choice of menu items and titles is controlled by the administrator of the server.
</p>
<style data-mw-deduplicate="TemplateStyles:r856303569">@media all and (max-width:720px){.mw-parser-output .tmulti>.thumbinner{width:100%!important;max-width:none!important}.mw-parser-output .tmulti .tsingle{float:none!important;max-width:none!important;width:100%!important;text-align:center}}</style><div class="thumb tmulti tnone center"><div class="thumbinner" style="width:561px;max-width:561px"><div class="tsingle" style="float:left;margin:1px;width:252px;max-width:252px"><div class="thumbimage"><a href="/wiki/File:Floodgap_gopher_top_menu.PNG" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/8/88/Floodgap_gopher_top_menu.PNG/250px-Floodgap_gopher_top_menu.PNG" width="250" height="182" srcset="//upload.wikimedia.org/wikipedia/en/8/88/Floodgap_gopher_top_menu.PNG 1.5x" data-file-width="369" data-file-height="269" /></a></div><div class="thumbcaption" style="clear:left">The top level menu of a Gopher server. Selecting the "Fun and Games" menu item...</div></div><div class="tsingle" style="float:left;margin:1px;width:305px;max-width:305px"><div class="thumbimage"><a href="/wiki/File:Floodgap_gopher_fun_menu.PNG" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/4/4c/Floodgap_gopher_fun_menu.PNG/303px-Floodgap_gopher_fun_menu.PNG" width="303" height="183" srcset="//upload.wikimedia.org/wikipedia/en/4/4c/Floodgap_gopher_fun_menu.PNG 1.5x" data-file-width="406" data-file-height="245" /></a></div><div class="thumbcaption" style="clear:left">...takes the user to the "Fun and Games" menu.</div></div><div style="clear:left"></div></div></div>
<p>Similar to a file on a Web server, a file on a Gopher server can be linked to as a menu item from any other Gopher server. Many servers take advantage of this inter-server linking to provide a directory of other servers that the user can access.
</p>
<h3><span class="mw-headline" id="Protocol">Protocol</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=6" title="Edit section: Protocol">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The Gopher protocol was first described in <a class="external mw-magiclink-rfc" rel="nofollow" href="https://tools.ietf.org/html/rfc1436">RFC 1436</a>. <a href="/wiki/Internet_Assigned_Numbers_Authority" title="Internet Assigned Numbers Authority">IANA</a> has assigned <a href="/wiki/Transmission_Control_Protocol" title="Transmission Control Protocol">TCP</a> <a href="/wiki/List_of_well-known_ports_(computing)" class="mw-redirect" title="List of well-known ports (computing)">port</a> 70 to the Gopher protocol.
</p><p>The protocol is simple to negotiate, making it possible to browse without using a client. A standard gopher session may therefore appear as follows:
</p>
<pre>/Reference
1CIA World Factbook     /Archives/mirrors/textfiles.com/politics/CIA    gopher.quux.org 70
0Jargon 4.2.0   /Reference/Jargon 4.2.0 gopher.quux.org 70      +
1Online Libraries       /Reference/Online Libraries     gopher.quux.org 70     +
1RFCs: Internet Standards       /Computers/Standards and Specs/RFC      gopher.quux.org 70
1U.S. Gazetteer /Reference/U.S. Gazetteer       gopher.quux.org 70      +
iThis file contains information on United States        fake    (NULL)  0
icities, counties, and geographical areas.  It has      fake    (NULL)  0
ilatitude/longitude, population, land and water area,   fake    (NULL)  0
iand ZIP codes. fake    (NULL)  0
i       fake    (NULL)  0
iTo search for a city, enter the city's name.  To search        fake    (NULL) 0
ifor a county, use the name plus County -- for instance,        fake    (NULL) 0
iDallas County. fake    (NULL)  0
</pre>
<p>Here, the client has established a TCP connection with the server on port 70, the standard gopher port. The client then sends a string followed by a carriage return followed by a <a href="/wiki/Newline" title="Newline">line feed</a> (a "CR + LF" sequence). This is the selector, which identifies the document to be retrieved. If the item selector were an empty line, the default directory would be selected. The server then replies with the requested item and closes the connection. According to the protocol, before the connection is closed, the server should send a full-stop (i.e., a period character) on a line by itself. However, as is the case here, not all servers conform to this part of the protocol and the server may close the connection without returning the final full-stop.
</p><p>In this example, the item sent back is a gopher menu, a directory consisting of a sequence of lines each of which describes an item that can be retrieved. Most clients will display these as <a href="/wiki/Hypertext" title="Hypertext">hypertext</a> links, and so allow the user to navigate through gopherspace by following the links.<sup id="cite_ref-wwwunleashed_5-1" class="reference"><a href="#cite_note-wwwunleashed-5">&#91;5&#93;</a></sup>
</p><p>All lines in a gopher menu are terminated by "CR + LF", and consist of five fields: the <i>item type</i> as the very first character (see below), the <i>display string</i> (i.e., the description text to display), a <i>selector</i> (i.e., a file-system pathname), <i>host name</i> (i.e., the domain name of the server on which the item resides), and <i>port</i> (i.e., the port number used by that server). The item type and display string are joined without a space; the other fields are separated by the tab character.
</p><p>Because of the simplicity of the Gopher protocol, tools such as <a href="/wiki/Netcat" title="Netcat">netcat</a> make it possible to download Gopher content easily from the command line:
</p>
<pre>echo jacks/jack.exe | nc gopher.example.org 70 &gt; jack.exe
</pre>
<p>The protocol is also supported by <a href="/wiki/CURL" title="CURL">cURL</a> as of 7.21.2-DEV.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">&#91;20&#93;</a></sup>
</p>
<h3><span class="mw-headline" id="Source_code_of_a_menu">Source code of a menu</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=7" title="Edit section: Source code of a menu">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Gopher menu items are defined by lines of <a href="/wiki/Tab-separated_values" title="Tab-separated values">tab-separated values</a> in a <a href="/wiki/Text_file" title="Text file">text file</a>. This file is sometimes called a <i>gophermap</i>. As the <a href="/wiki/Source_code" title="Source code">source code</a> to a gopher menu, a gophermap is roughly analogous to an <a href="/wiki/HTML" title="HTML">HTML</a> file for a <a href="/wiki/Web_page" title="Web page">web page</a>. Each tab-separated line (called a <i>selector line</i>) gives the <a href="/wiki/Client_software" class="mw-redirect" title="Client software">client software</a> a description of the menu item: what it is, what it's called, and where it leads. The client displays the menu items in the order that they appear in the gophermap.
</p><p>The first character in a selector line indicates the <i>item type</i>, which tells the client what kind of file or protocol the menu item points to. This helps the client decide what to do with it. Gopher's item types are a more basic precursor to the <a href="/wiki/Media_type" title="Media type">media type</a> system used by the Web and <a href="/wiki/Email_attachment" title="Email attachment">email attachments</a>.
</p><p>The item type is followed by the <i>user display string</i> (a description or label that represents the item in the menu); the selector (a <a href="/wiki/Path_(computing)" title="Path (computing)">path</a> or other string for the resource on the server); the <i>hostname</i> (the <a href="/wiki/Domain_name" title="Domain name">domain name</a> or <a href="/wiki/IP_address" title="IP address">IP address</a> of the server), and the <a href="/wiki/Network_port" class="mw-redirect" title="Network port">network port</a>.
</p><p>For example: The following selector line generates a link to the "/home" <a href="/wiki/Directory_(computing)" title="Directory (computing)">directory</a> at the <a href="/wiki/Subdomain" title="Subdomain">subdomain</a> gopher.floodgap.com, on <a href="/wiki/Port_(computer_networking)" title="Port (computer networking)">port</a> 70. The item type of <var style="padding-right: 1px;">1</var> indicates that the resource is a Gopher menu. The string "Floodgap Home" is what the user  sees in the menu.
</p>
<pre>1Floodgap Home	/home	gopher.floodgap.com	70
</pre>

<p>In addition to selector lines, a gophermap may contain <i>comment lines</i>. Comment lines are not for <a href="/wiki/Code_comments" class="mw-redirect" title="Code comments">code comments</a>; rather, they are lines of text sent to the client to display alongside the menu items, such as for a menu description or welcome message. A comment line contains no <a href="/wiki/Tab_character" class="mw-redirect" title="Tab character">tab characters</a>.
</p>
<h4><span class="mw-headline" id="Item_types">Item types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=8" title="Edit section: Item types">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>In a Gopher menu's source code, a one-character code indicates what kind of content the client should expect. This code may either be a digit or a letter of the alphabet; letters are <a href="/wiki/Case-sensitive" class="mw-redirect" title="Case-sensitive">case-sensitive</a>.
</p><p>The <a href="/wiki/Technical_specification" class="mw-redirect" title="Technical specification">technical specification</a> for Gopher, <a class="external mw-magiclink-rfc" rel="nofollow" href="https://tools.ietf.org/html/rfc1436">RFC 1436</a>, defines 14 item types. A one-character code indicates what kind of content the client should expect. Item type <code class="mw-highlight" id="" style="" dir="ltr">3</code> is an <a href="/wiki/Error_code" class="mw-redirect" title="Error code">error code</a> for <a href="/wiki/Exception_handling" title="Exception handling">exception handling</a>. Gopher client authors improvised item types <code class="mw-highlight" id="" style="" dir="ltr">h</code> (HTML), <code class="mw-highlight" id="" style="" dir="ltr">i</code> (informational message), and <code class="mw-highlight" id="" style="" dir="ltr">s</code> (<a href="/wiki/Sound_file" class="mw-redirect" title="Sound file">sound file</a>) after the publication of <a class="external mw-magiclink-rfc" rel="nofollow" href="https://tools.ietf.org/html/rfc1436">RFC 1436</a>.
</p>

<h4><span class="mw-headline" id="URL_links">URL links</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=9" title="Edit section: URL links">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Historically, to create a link to a Web server, "GET /" was used as a pseudo-selector to emulate an <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a> <a href="/wiki/GET_request" class="mw-redirect" title="GET request">GET request</a>. <a href="/wiki/John_Goerzen" title="John Goerzen">John Goerzen</a> created an addition<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">&#91;21&#93;</a></sup> to the Gopher protocol, commonly referred to as "<a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">URL</a> links", that allows links to any protocol that supports URLs. For example, to create a link to <a rel="nofollow" class="external free" href="http://gopher.quux.org/">http://gopher.quux.org/</a>, the item type is <code class="mw-highlight" id="" style="" dir="ltr">h</code>, the display string is the title of the link, the item selector is "URL:http://gopher.quux.org/", and the domain and port are that of the originating Gopher server (so that clients that do not support URL links will query the server and receive an HTML redirection page).
</p>
<h3><span class="mw-headline" id="Related_technology">Related technology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=10" title="Edit section: Related technology">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The master Gopherspace search engine is <a href="/wiki/Veronica_(search_engine)" title="Veronica (search engine)">Veronica</a>. Veronica offers a keyword search of all the public Internet Gopher server menu titles. A Veronica search produces a menu of Gopher items, each of which is a direct pointer to a Gopher data source. Individual Gopher servers may also use localized search engines specific to their content such as <a href="/wiki/Jughead_(search_engine)" title="Jughead (search engine)">Jughead</a> and <a href="/wiki/Jughead_(search_engine)" title="Jughead (search engine)">Jugtail</a>.
</p><p><a href="/wiki/GopherVR" title="GopherVR">GopherVR</a> is a 3D virtual reality variant of the original Gopher system.
</p>

<p>Browsers that do not natively support Gopher can still access servers using one of the available Gopher to <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a> gateways.
</p><p>Gopher support was disabled in <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> versions 5.x and 6 for Windows in August 2002 by a patch meant to fix a security vulnerability in the browser's Gopher protocol handler to reduce the attack surface which was included in IE6 SP1; however, it can be re-enabled by editing the <a href="/wiki/Windows_registry" class="mw-redirect" title="Windows registry">Windows registry</a>. In <a href="/wiki/Internet_Explorer_7" title="Internet Explorer 7">Internet Explorer 7</a>, Gopher support was removed on the <a href="/wiki/WinINET" class="mw-redirect" title="WinINET">WinINET</a> level.<sup id="cite_ref-31" class="reference"><a href="#cite_note-31">&#91;31&#93;</a></sup>
</p>
<h4><span class="mw-headline" id="Gopher_browser_plug-ins">Gopher browser plug-ins</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=13" title="Edit section: Gopher browser plug-ins">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>For <a href="/wiki/Mozilla_Firefox" class="mw-redirect" title="Mozilla Firefox">Mozilla Firefox</a> and <a href="/wiki/SeaMonkey" title="SeaMonkey">SeaMonkey</a>, OverbiteFF<sup id="cite_ref-floodgap1_32-0" class="reference"><a href="#cite_note-floodgap1-32">&#91;32&#93;</a></sup> extends Gopher browsing and supports the current versions of the browsers (<a href="/wiki/Firefox_38" class="mw-redirect" title="Firefox 38">Firefox 38</a> until Firefox 56, and equivalent versions of SeaMonkey). (There is also a version of OverbiteFF available that supports <a href="/wiki/Firefox_3.6" title="Firefox 3.6">Firefox 3.6</a> and <a href="/wiki/Firefox_4" title="Firefox 4">4</a> to <a href="/wiki/Firefox_37" class="mw-redirect" title="Firefox 37">37</a>, and equivalent versions of SeaMonkey.) For the new Firefox Quantum, OverbiteWX was released. It includes support for accessing Gopher servers not on port 70 using a whitelist and for <a href="/wiki/CCSO_Nameserver" title="CCSO Nameserver">CSO/ph queries</a>, and allows versions of Firefox and SeaMonkey that do not support Gopher natively to access Gopher servers. Plugins are also available for <a href="/wiki/Konqueror" title="Konqueror">Konqueror</a>,<sup id="cite_ref-33" class="reference"><a href="#cite_note-33">&#91;33&#93;</a></sup> in the past a proxy-based extension for <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a> was available but is no longer maintained and does not work with the current releases.<sup id="cite_ref-floodgap1_32-1" class="reference"><a href="#cite_note-floodgap1-32">&#91;32&#93;</a></sup>
</p>
<h3><span class="mw-headline" id="Gopher_clients_for_mobile_devices">Gopher clients for mobile devices</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=14" title="Edit section: Gopher clients for mobile devices">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Some<sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Manual_of_Style/Words_to_watch#Unsupported_attributions" title="Wikipedia:Manual of Style/Words to watch"><span title="The material near this tag possibly uses too-vague attribution or weasel words. (January 2014)">who?</span></a></i>&#93;</sup> have suggested that the bandwidth-sparing simple interface of Gopher would be a good match for mobile phones and <a href="/wiki/Personal_digital_assistants" class="mw-redirect" title="Personal digital assistants">personal digital assistants</a> (PDAs),<sup id="cite_ref-34" class="reference"><a href="#cite_note-34">&#91;34&#93;</a></sup> but so far, mobile adaptations of <a href="/wiki/HTML" title="HTML">HTML</a> and <a href="/wiki/XML" title="XML">XML</a> and other simplified content have proven more popular. The <a href="/wiki/PyGopherd" title="PyGopherd">PyGopherd</a> server provides a built-in <a href="/wiki/Website_Meta_Language" title="Website Meta Language">WML</a> front-end to Gopher sites served with it.
</p><p>The early 2010s saw a renewed interest in native Gopher clients for popular <a href="/wiki/Smartphones" class="mw-redirect" title="Smartphones">smartphones</a>: Overbite, an open source client for <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android 1.5+</a> was released in <a href="/wiki/Alpha_stage" class="mw-redirect" title="Alpha stage">alpha stage</a> in 2010.<sup id="cite_ref-ars_35-0" class="reference"><a href="#cite_note-ars-35">&#91;35&#93;</a></sup> PocketGopher was also released in 2010, along with its <a href="/wiki/Source_code" title="Source code">source code</a>, for several <a href="/wiki/Java_Platform,_Micro_Edition" title="Java Platform, Micro Edition">Java ME</a> compatible devices. iGopher was released in 2011 as a <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary</a> client for <a href="/wiki/IPhone" title="IPhone">iPhone</a> and <a href="/wiki/IPad" title="IPad">iPad</a> devices, however it's no longer maintained and can't be used on iOS 11.
</p>
<h3><span class="mw-headline" id="Other_Gopher_clients">Other Gopher clients</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=15" title="Edit section: Other Gopher clients">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Gopher popularity was at its height at a time when there were still many equally competing computer architectures and operating systems. As a result, there are several Gopher clients available for <a href="/wiki/Acorn_Computers_Ltd#Network_Computers" class="mw-redirect" title="Acorn Computers Ltd">Acorn RISC OS</a>, <a href="/wiki/AmigaOS" title="AmigaOS">AmigaOS</a>, <a href="/wiki/MiNT" title="MiNT">Atari MiNT</a>, <a href="/wiki/Conversational_Monitor_System" title="Conversational Monitor System">CMS</a>, <a href="/wiki/DOS" title="DOS">DOS</a>, <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">classic Mac OS</a>, <a href="/wiki/MVS" title="MVS">MVS</a>, <a href="/wiki/NeXT" title="NeXT">NeXT</a>, <a href="/wiki/OS/2_Warp" class="mw-redirect" title="OS/2 Warp">OS/2 Warp</a>, most <a href="/wiki/Unix-like_operating_system" class="mw-redirect" title="Unix-like operating system">UNIX-like</a> operating systems, <a href="/wiki/OpenVMS" title="OpenVMS">VMS</a>, <a href="/wiki/Windows_3.1x" title="Windows 3.1x">Windows 3.x</a>, and <a href="/wiki/Windows_9x" title="Windows 9x">Windows 9x</a>. <a href="/wiki/GopherVR" title="GopherVR">GopherVR</a> was a client designed for 3D visualization, and there is even a Gopher client in <a href="/wiki/MOO" title="MOO">MOO</a>.<sup id="cite_ref-36" class="reference"><a href="#cite_note-36">&#91;36&#93;</a></sup><sup id="cite_ref-37" class="reference"><a href="#cite_note-37">&#91;37&#93;</a></sup> The majority of these clients are <a href="/wiki/Hard_coding" title="Hard coding">hard-coded</a> to work on <a href="/wiki/Transmission_Control_Protocol" title="Transmission Control Protocol">TCP</a> <a href="/wiki/List_of_well-known_ports_(computing)" class="mw-redirect" title="List of well-known ports (computing)">port</a> 70.
</p>
<h3><span class="mw-headline" id="Gopher_to_HTTP_gateways">Gopher to HTTP gateways</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gopher_(protocol)&amp;action=edit&amp;section=16" title="Edit section: Gopher to HTTP gateways">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Users of Web browsers that have incomplete or no support for Gopher can access content on Gopher servers via a server gateway or <a href="/wiki/Proxy_server" title="Proxy server">proxy server</a> that converts Gopher menus into <a href="/wiki/HTML" title="HTML">HTML</a>; known proxies are the Floodgap Public Gopher proxy and Gopher Proxy. Similarly, certain server packages such as GN and <a href="/wiki/PyGopherd" title="PyGopherd">PyGopherd</a> have built-in Gopher to <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a> interfaces. <a href="/wiki/Squid_(software)" title="Squid (software)">Squid Proxy</a> software gateways any gopher:// URL to HTTP content, enabling any browser or web agent to access gopher content easily.
</p>
